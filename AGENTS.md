# AGENTS.md

`gmt-engine` is a generic application engine extracted from GMT (the stable
fractal explorer at `h:/GMT/workspace-gmt/stable/`). It provides DDFS
(Data-Driven Feature System), animation, UI primitives, save/load, shortcuts,
undo, a worker contract, and plugin seams. Apps such as `app-gmt` (the GMT
re-port), `fluid-toy`, and `fractal-toy` install features and core plugins on
top of it.

This file follows the [agents.md](https://agents.md/) convention: a thin
entry-point for AI agents and humans. It does not duplicate `CLAUDE.md` — it
points at it.

## Reading order for AI agents

When you boot fresh on this repo, read in this order. Stop after step 3 unless
your task pulls you deeper.

1. **`CLAUDE.md`** — the rules. Critical: the "Read first" table, the
   architecture rules (feature isolation, render loop ownership, frozen
   registry), and the anti-pattern list.
2. **`CODEBASE_MAP.md`** — orientation. What's in `engine/` vs `engine-gmt/`
   vs `app-gmt/` vs `fluid-toy/`.
3. **`docs/DOCS_INDEX.md`** — full TOC. Pick the specific doc for the area
   you're about to touch.
4. **`docs/modules/<area>/<file>.md`** (when present) — per-file deep dives
   with frontmatter (source SHA, public API, dependencies). Generated by the
   2026-05-19 audit. Authoritative for the file they cite.
5. The source itself. Always cite `file:line` when reporting findings.

**Before you bulk-load source for a subsystem, run a load plan:**
`npm run context:cost -- <subsystem|tier|path|app:name>`. It tells you the
cheapest docs-first path and its token cost, and flags heavy files to read in
sections. See [`docs/policy/context-loading-protocol.md`](docs/policy/context-loading-protocol.md).

## Conventions

Lifted briefly from `CLAUDE.md` — read that file for the full version.

- **Features are isolated.** State lives at `store[featureId]`. Cross-feature
  reads require `dependsOn`. Bridges or derived values for coordination, never
  store reach-through. (`docs/engine/02_Feature_Registry.md`,
  `docs/engine/09_Bridges_and_Derived.md`)
- **Render loop is app-owned.** Engine exposes `TickRegistry` phases; the app
  (or `@engine/render-loop` core plugin) calls `runTicks(dt)`. Don't bypass
  with ad-hoc `useFrame`. (`docs/engine/01_Architecture.md`)
- **Feature registry freezes at store construction.** All
  `featureRegistry.register()` calls happen before `createEngineStore()`. Late
  registration throws in dev. (`docs/engine/03_Plugin_Contract.md`)
- **Every DDFS param is animatable + undoable by construction.** No
  per-feature wiring. (`docs/engine/08_Animation.md`)
- **TypeScript `isolatedModules: true`.** Cross-module type-only re-exports
  MUST use `export type { X }` / `import type { X }`. Otherwise esbuild leaves
  the export in JS and you get a runtime SyntaxError.
- **No `as any` without a comment.** Type-grafts (engine-core stub vs
  engine-gmt real) sometimes need them; document why.
- **Don't edit `docs/gmt/`.** Pre-extraction reference. Engine-scope changes
  go in `docs/engine/` or `docs/modules/`.
- **Citations.** Every architectural claim cites `path/to/file.ts:N` or
  `path/to/file.ts:N-M`. If you can't cite, you don't know it yet.

## Automated checks

- `npm run typecheck` — `tsc`, exits 0.
- `npm run orphans` — `knip`; unused-file scan (real import-graph walk).
- `npm run smoke:boot` — headless Chromium boot, fails on pageerrors.
- `npm run smoke:interact` — state-flow + preset round-trip.
- `npm run smoke:screenshot` — visual baseline to `debug/scratch/`.
- `npm run context:map` — rebuild the token-cost map of the repo.
- `npm run context:cost -- <target>` — print a cheapest-first load plan.
- `npm run context:check` — fail if the cost map is stale or a file is unclassified.
