Production mesh exporter unification — planned, not yet started. Supersampled 2026-04-03 using 4 deep-dive research agents + 2 independent architect agents who derived plans from raw facts only.

**Why:** The mesh export has a parallel shader builder (`SDFShaderBuilder.ts`) and independent store (`meshExportStore`) that must manually replicate every main app feature. Interlace, shared transforms, preamble ordering bugs are symptoms of this drift. Every new feature requires a separate manual port to the mesh path.

**Goal:** Route mesh SDF generation through the existing DDFS feature injection pipeline (`ShaderFactory` + `ShaderBuilder` + feature `inject()` hooks) so new features are automatically free in mesh export.

---

## High-Confidence Core Plan (all 3 independent derivations agree)

### Part 1 — Add 'Mesh' to RenderVariant

```typescript
type RenderVariant = 'Main' | 'Physics' | 'Histogram' | 'Mesh';
```

One-line type change. No behavior change. All subsequent code gates on this.

### Part 2 — ShaderFactory: generateMeshSDFShader()

```typescript
static generateMeshSDFShader(config: ShaderConfig): string
```

Routes through `buildShader(config, 'Mesh')` — same pattern as Physics/Histogram variants. Calls all feature `inject(builder, config, 'Mesh')` hooks in topological order. Returns `builder.buildMeshSDF()`.

Features that inject into stages 11–17 (lighting, materials, ray gen, post-processing) simply inject into buffers that `buildMeshSDF()` ignores. No changes needed to those features.

### Part 3 — ShaderBuilder: buildMeshSDF()

New terminal method. Assembles only stages 1–10 (uniforms, preamble, functions, DE loop) and wraps in mesh entry point:

```glsl
float formulaDE(vec3 pos) {
    return mapDist(pos);   // mapDist() returns float — cleaner than map(pos).x
}
```

**Critical prerequisite**: verify `mapDist()` generated by DE_MASTER() is a pure function of `vec3 p` with no dependencies on raymarcher globals. If it is not pure, the wrapper won't work and we need a different approach (see Risk 6 below).

### Part 4 — Config Translation Layer

meshExportStore holds `MeshExportState`, not `ShaderConfig`. A translation helper is needed:

```typescript
function buildMeshShaderConfig(state: MeshExportState): ShaderConfig
```

This constructs:
- `formula`: from `state.selectedFormulaId`
- `coreMath`: from `state.formulaParams` (paramA-F, vec2/vec3/vec4, julia, juliaMode, escapeThresh)
- `interlace`: from `state.interlaceState` — **must explicitly set `interlaceCompiled: true`** or interlace silently gets skipped by `Interlace.inject()`
- `geometry`: defaults (mesh export doesn't expose geometry transforms in UI)
- `pipelineRevision: 0`, `formula`, other required ShaderConfig fields

### Part 5 — GPU Pipeline: SDF Passes Only

In `gpu-pipeline.ts`, replace SDF pass shader generation:

```
Before: buildMeshSDFShader({ definition, deType, deSamples, interlace })
After:  ShaderFactory.generateMeshSDFShader(buildMeshShaderConfig(storeState))
```

### Part 6 — Non-SDF Passes: Keep As-Is

The 4 non-SDF passes are fixed-function compute shaders with no feature injection:
- `buildMeshEscapeShader()` — orbit escape detection
- `buildMeshNewtonShader()` — vertex refinement (MRT output)
- `buildMeshColorShader()` — orbit-trap coloring (may need pass-type discrimination)
- `buildMeshPreviewShader()` — raymarched 3D preview

These stay in a reduced `SDFShaderBuilder.ts` (or rename to `MeshPassBuilders.ts`). Do not route them through ShaderFactory — they need different output structures that no existing feature injects.

### Part 7 — meshExportStore: No Changes Needed

Keep all formula-related state: `selectedFormulaId`, `loadedDefinition`, `formulaParams`, `interlaceState`. The mesh export app is a separate window from the main app — no shared memory, no fractalStore wiring possible. The translation layer (Part 4) bridges the formats.

### Part 8 — UI: No Changes Needed

`FormulaSelector.tsx` stays — it populates meshExportStore correctly. No integration into main app panel needed. The mesh export remains a separate tab.

**Verdict**: Rejected. UI integration is unnecessary scope. The existing FormulaSelector works correctly.

## Risks (identified by independent agents, not in original plan)

### Risk 1 — Rotation stub conflict (HIGH)
`MESH_GLSL_HELPERS` has empty `applyPreRotation/applyPostRotation/applyWorldRotation` stub functions. If `Geometry.inject()` is called with variant `'Mesh'`, it calls `builder.setRotation(enabled)` and injects `SHARED_TRANSFORMS_GLSL` which may declare conflicting implementations.

**Resolution options:**
- Option A (preferred): Remove rotation stubs from MESH_GLSL_HELPERS. Let geometry feature control rotation for mesh too — makes rotation actually work in mesh export.
- Option B (safe): Guard in `Geometry.inject()`: when `variant === 'Mesh'`, emit stub declarations instead of full rotation code.

### Risk 2 — Uniform duplication (HIGH)
`MESH_GLSL_UNIFORMS` declares `uParamA-F`, `uVec2A-C` etc. `CoreMath.inject()` will also call `builder.addUniform()` for those same names → duplicate GLSL declarations → compile failure.

**Resolution**: Drop `MESH_GLSL_UNIFORMS` from MESH_GLSL_HELPERS. Rely entirely on `addUniform()` calls from feature `inject()` hooks. Verify every uniform currently in MESH_GLSL_UNIFORMS is covered by some feature's inject().

### Risk 3 — interlaceCompiled flag (MEDIUM)
`Interlace.inject()` guards on `if (!state?.interlaceCompiled) return`. The translation layer must explicitly set `interlace.interlaceCompiled: true` when `meshExportStore.interlaceState` is non-null. If this flag is missing, interlace silently does nothing.

**Resolution**: In `buildMeshShaderConfig()`, set `interlaceCompiled: true` when interlaceState is present.

### Risk 4 — GMF-embedded custom definitions (MEDIUM)
If a formula was loaded from a GMF file with a definition not in the main formula registry (e.g. a Workshop-created formula), `ShaderFactory`'s `registry.get(formula)` will return undefined and shader generation will fail.

**Resolution**: In `buildMeshShaderConfig()` or the GPU pipeline setup, pre-register the `loadedDefinition` in the formula registry before calling generateMeshSDFShader.

### Risk 5 — Color/Preview passes need pass-specific GLSL (MEDIUM)
Color pass needs orbit-trap coloring code. Preview pass needs a full raymarcher. These are not things any existing feature injects. They may require `MeshPassType` discrimination in ShaderBuilder or a separate injection mechanism.

**Resolution**: Keep these passes in the old SDFShaderBuilder. Do not route them through ShaderFactory unless features are extended with mesh-pass-specific injection.

### Risk 6 — mapDist() purity (MEDIUM, verify before coding)
The plan to wrap `mapDist()` as `formulaDE()` assumes mapDist is a pure function of `vec3 p` with no dependencies on raymarcher global state. Verify against `shaders/chunks/de.ts` that mapDist doesn't reference any global variables injected by the raymarcher before writing the wrapper.

**If mapDist is not pure**: Use `map(pos).x` instead (map returns vec4, take .x = distance). Or inline the DE loop body directly in formulaDE without using either wrapper.

## Implementation Sequence

1. Verify `mapDist()` purity in de.ts (before any coding)
2. Add `'Mesh'` to `RenderVariant` type
3. Add `buildMeshSDF()` to ShaderBuilder — stages 1–10 only + formulaDE() wrapper
4. Add `generateMeshSDFShader(config)` to ShaderFactory — wires through feature inject loop
5. Resolve rotation stub conflict (Risk 1) — decide Option A or B, implement
6. Resolve uniform duplication (Risk 2) — remove MESH_GLSL_UNIFORMS from MESH_GLSL_HELPERS
7. Write `buildMeshShaderConfig(state: MeshExportState): ShaderConfig`
8. Handle custom formula registration (Risk 4) in the config builder or pipeline setup
9. Replace SDF pass shader generation in gpu-pipeline.ts with new path
10. Verify output matches previous behavior with Mandelbulb, interlaced formula, polyhedral formulas
11. Remove `buildMeshSDFShader()` and its interlace helpers from SDFShaderBuilder.ts
12. Leave escape/Newton/color/preview passes untouched in SDFShaderBuilder (or renamed file)
