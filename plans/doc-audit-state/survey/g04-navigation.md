## Public API surface

- Barrel exports `GmtNavigation` (default-Navigation) and `GmtNavigationHud` (default-HudOverlay) plus types `CameraMode`, `CameraState`, `PreciseVector3` re-exported from `../types` — `engine-gmt/navigation/index.ts:22-24`.
- Module is documented as a verbatim port of GMT navigation (path-rewrites only, zero logic change) — `engine-gmt/navigation/index.ts:1-20`.
- `GmtNavigation` props: `mode: CameraMode`, optional `onStart(s: CameraState)` and `onEnd()` callbacks, `hudRefs` (container/speed/dist/reset/reticle), `setSceneOffset(v)`, `fitScale = 1.0` — `Navigation.tsx:120-133`.
- `GmtNavigationHud` props: `isMobile`, optional `activeHint`/`onDismissHint` (typed `ActiveHint = unknown`, hint system not ported — comment `HudOverlay.tsx:7-11`), `hudRefs`, `region: 'top'|'bottom'|'all'` — `HudOverlay.tsx:13-30`.
- Internal hooks (not exported from barrel): `useInputController(mode, speed, setSpeed, hudRefs?)` returns `{ moveState, isDraggingRef, dragStart, mousePos, speedRef, joystickMove, joystickLook, invertY, rollVelocity, isInteracting }` — `useInputController.ts:15-19, 264`; `usePhysicsProbe(hudRefs, speedRef)` returns `{ distAverageRef }` — `usePhysicsProbe.ts:12-19, 172`.
- Modifier helpers: `CAMERA_BOOST_MULT = 4.0`, `CAMERA_PRECISE_MULT = 0.1`, `getCameraModifier(boost, precise)`, `getCameraModifierFromEvent(e)` — `modifiers.ts:8-19`.

## Architecture

- **Two-mode design (Orbit, Fly) share one camera/state.** Header comment justifies the single-file size (~20 shared refs across modes) — `Navigation.tsx:6-19`. Outside Orbit the JSX returns `null` and only refs + `useFrame` run — `Navigation.tsx:1351`.
- **Unified coordinate system** = `sceneOffset + camera.position`. Steady-state camera sits at origin; mid-gesture `camera.position` accumulates motion and gets absorbed back into `sceneOffset` to preserve f32 mantissa headroom at deep zoom — `Navigation.tsx:21-30`.
- **Canonical absorb path** = `absorbOrbitPosition(silent, keepTarget)` — `Navigation.tsx:265-325`. `silent=true` queues atomic offset via `engine.queueOffsetSync(...)` and writes store directly (no accumulation reset) for orbit/wheel/middle/pan end; `silent=false` uses `setSceneOffset(...)` (full sync + accum reset) for mode switches. `keepTarget=true` shifts `OrbitControls.target` by `-camera.position` to preserve its world position (used only by per-frame pan absorb) — `Navigation.tsx:258-264, 297-309`.
- **Absorb-gen race guard** = `absorbGenRef` increments on every absorb (`Navigation.tsx:273`) so any in-flight hover pick (issued against old `sceneOffset`) drops its result on resolution at `Navigation.tsx:557`. Symptom this fixes: "quicker the interaction → more likely glitch" — `Navigation.tsx:197-204`.
- **Cursor-anchored gesture pivots** live in two refs: `hoverPivotWorldRef` (WORLD-space, stable across gestures; `null` = pick miss) and `gestureActivePivotRef` (LOCAL-space snapshot captured at gesture start, held for the gesture). `snapshotHoverPivotLocal()` converts world→local using the CURRENT `sceneOffset` at gesture start — `Navigation.tsx:175-227`.
- **Effect roster (source order):** (1) mount init `Navigation.tsx:353-376`; (2) teleport/transition listener `:379-471`; (3) hover pre-pick `:523-563`; (4) wheel zoom `:578-657`; (5) custom left-drag orbit `:673-831`; (6) middle-drag dolly `:837-912`; (7) pivot reticle DOM dot `:918-943`; (8) orbit pointer-gate `:946-976`; (9) camera-lock effect `:981-987`; (10) mode-switch `useLayoutEffect` `:990-1014`; (11) per-frame integrator `:1016-1349`. Header comment lists this in different language at `Navigation.tsx:51-72`.
- **Hover pre-pick** (Orbit, pointermove on `gl.domElement`): self-throttling in-flight gate (`hoverPickInFlightRef`) plus 10 px movement gate (`< 100` squared px) — `Navigation.tsx:537-547`. Pick result is written to `hoverPivotWorldRef` only if `absorbGenRef` is unchanged since issue — `Navigation.tsx:548-559`. Skips during ANY active gesture (`isOrbitDragging`, `isScrollingRef`, `isCustomOrbitRef`) per `:531-536`.
- **Custom wheel handler** = TRANSLATION along cursor→pivot ray: `new_pos = pivot + (camera - pivot) * f`, `f = exp(deltaY * 0.0015 * dollySpeed)` — `Navigation.tsx:610-614`. Bypasses drei's `DOLLY` which would `lookAt(target)` and cause lateral re-orientation — `Navigation.tsx:565-577`. Per-event absorb keeps `camera.position` near zero between events; pivot is shifted by `-camera.position` into the new local frame before reset — `Navigation.tsx:615-626`. Burst end is a 100 ms `scrollEndTimeout` that does cleanup-only (no offset mutation) — `Navigation.tsx:644-650`.
- **Custom left-drag orbit** = Blender-style "rotate around cursor": rotate `(camera.position - pivot)` AND `camera.quaternion` by the same composite quaternion so the pivot's direction-from-camera is invariant in camera-space — `Navigation.tsx:659-672`. Azimuth uses frozen `gestureUpRef` (snapshotted at pointerdown so Q/E roll between gestures is honoured but within-gesture pole stays put) — `Navigation.tsx:498-500, 720, 764-776`. Polar clamped to `[eps, π-eps]` (`eps = 0.02`) — `Navigation.tsx:779-784`. Synchronous application in `pointermove` not `useFrame` — to avoid 1-frame latency — `Navigation.tsx:754-761`. Touch (`pointerType === 'touch'`) cedes to drei's native `THREE.TOUCH.ROTATE` — `Navigation.tsx:684-686`.
- **Middle-drag dolly** uses same translation math as wheel but driven by vertical drag (`dy * 0.005`, drag-down = zoom-in, Blender convention) — `Navigation.tsx:861-866`. Same per-event absorb + on-up second absorb — `Navigation.tsx:883-902`.
- **Custom-vs-drei gating** is a per-frame computation at `Navigation.tsx:1278-1285`: when `orbitCursorAnchor` is on, drei is enabled ONLY for PAN (right-drag) while custom handlers own ROTATE/DOLLY/MIDDLE (drei's `mouseButtons` set to `LEFT: -1, MIDDLE: -1, RIGHT: PAN`; `enableZoom={!cursorAnchorEffective}` at `:1368-1371`). When off, drei owns all of ROTATE/DOLLY/PAN.
- **Cursor-anchor force-disabled on mobile** regardless of saved setting — multi-touch path is drei's responsibility and the per-event hover machinery is wasted there — `Navigation.tsx:157-162` (`cursorAnchorEffective = !isMobile && (navSettings?.orbitCursorAnchor ?? true)`).
- **Pivot-marker reticle** is an imperatively-created DOM `<div>` appended to `gl.domElement.parentElement` in an Orbit-only effect (`Navigation.tsx:918-943`) and positioned each frame via `camera.project()` in `useFrame` (`:1107-1147`). Active-gesture: full opacity over `gestureActivePivotRef`; idle hover: 0.55 opacity over `hoverPivotWorldRef` re-localised each frame; off-screen/behind-camera: hidden.
- **Teleport / transition event listeners** subscribe to `FractalEvents.on('camera_teleport', ...)` and `on('camera_transition', ...)` — `Navigation.tsx:468-470`. Teleport sets camera + offset directly (`engine.shouldSnapCamera`, `engine.dirty`), reseeds `distAverageRef` / `orbitRadiusRef`, and re-syncs orbit target preserving roll via `camera.up` rebuild — `Navigation.tsx:380-424`. Transition lerps unified position + slerps rotation over 0.5 s, then emits a final `camera_teleport` to commit exact precision — `Navigation.tsx:427-470, 1037-1077`.
- **Mode-switch (`useLayoutEffect`):** emits `camera_snap`, force-absorbs residual orbit position (mid-scroll switch safety), seeds Fly speed from `engine.lastMeasuredDistance` or orbit radius, or reseats orbit pivot via `initOrbitPivot()` — `Navigation.tsx:990-1014`.
- **Camera lock** (`isCameraLocked`) = `isPlaying && (!isRecording || !recordCamera) && any tracks[k] startsWith('camera.')` OR `isScrubbing` — `Navigation.tsx:978`. While locked: drei disabled, `setIsCameraInteracting(false)`, `engine.cameraInUse = false`, early return from `useFrame` — `Navigation.tsx:1079-1087`.
- **Per-frame integrator** order: transition lerp → camera-lock early-return → orbit-ready re-init → pos/rot delta detect → pivot reticle project → `isCameraInteracting` push → accumulation reset (`engine.dirty`) → keyframe capture if recording → stop-debounce (100 ms) that normalises camera→origin and writes `cameraRot`/`sceneOffset`/`targetDistance` to store (skipping sky-sentinel `MISS_DIST = 1000.0`) — `Navigation.tsx:1016-1220`.
- **Fly branch (`mode === 'Fly'`)** delegates physics to `flyController = new CameraController()` (imported from `engine-gmt/engine/controllers/CameraController` — outside g04 scope) — `Navigation.tsx:117, 169, 1242-1258`. Drag delta is scaled by `fitScale` to prevent sensitivity amplification under fixed-resolution mode — `Navigation.tsx:1226-1228`. Reticle DOM ref shows drag offset as a translate3d on the inertial reticle marker — `Navigation.tsx:1230-1240`.
- **Orbit branch tail** does per-frame Q/E roll application — applies `applyAxisAngle(fwd, rollAngle)` to `camera.up` (and to `gestureUpRef` if mid custom-orbit so the per-event up reset doesn't strip roll), skipping the manual `update()` when drei is already running (avoids `(1 - dampingFactor)` double-multiply staggered motion) — `Navigation.tsx:1307-1321`. Per-frame pan absorb (`keepTarget=true`) runs while `isOrbitDragging` and `camera.position.lengthSq() > 1e-8`, then sets `engine.dirty` because zero `camera.position` masks pan motion from the `posChanged` check — `Navigation.tsx:1323-1337`.
- **`unifiedOffsetRef` shadow** maintained at end of every `useFrame` as `sceneOffset + camera.position` so any consumer sees canonical world position even mid-drag — `Navigation.tsx:1340-1348`.
- **OrbitControls config** at JSX render: `makeDefault`, `enableDamping={false}`, `target={orbitTargetZero.current}`, touch bindings (`ONE: THREE.TOUCH.ROTATE`, `TWO: THREE.TOUCH.DOLLY_PAN`), and onStart/onEnd that seat `orbitRef.current.target` at `camera.position + forward × distAverageRef.current` so drei's PAN distance calculation isn't zero — `Navigation.tsx:1353-1400`.
- **`useInputController`** owns keyboard/mouse/wheel/joystick state. Move state has 10 booleans (forward, backward, left, right, up, down, rollLeft, rollRight, boost, precise) — `useInputController.ts:9-13`. Key bindings: WASD translate, QE roll, Space up, KeyC down, ShiftL/R = boost, AltL/R = precise. Prevents browser defaults for Ctrl+W, Space, Alt — `useInputController.ts:67-101`. Ignores keys when target is `INPUT`/`TEXTAREA`/`contentEditable`/`.cm-editor`, and when `isTimelineHovered` — `:68-78`.
- **Roll integration in input controller** runs in its own `useFrame` (priority default): exponential approach to target with `accelRate = 1.0` while held, `3.0` decaying; scaled by `max(0.1, speedRef.current)` so Q/E retains usability at low fly speed; zeroes below 0.001 threshold — `useInputController.ts:53-64`.
- **Wheel handling** (`useInputController.ts:125-163`): respects `selectMovementLock`. Ortho mode (`Math.abs(state.optics.camType - 1.0) < 0.1`) adjusts `state.optics.orthoScale` directly with `(1 + dir * 0.1 * mod)`, clamped `[1e-10, 1000]`. Fly mode adjusts `flySpeed` with step `0.005` (<0.05), `0.01` default, `0.02` (>0.1), clamped `[0.001, 1.0]`. Orbit mode only `markActivity()` because the Navigation custom wheel handler owns the real zoom math.
- **Fly drag origin** (`useInputController.ts:200-225`): mouse-down on canvas captures NDC dragStart; mobile + Fly is suppressed (`isMobile && mode === 'Fly'` returns) since joystick is the mobile path; HUD/`pointer-events-auto` click are excluded via `hudRefs.contains` or class fallback.
- **Orbit drag activity** (`useInputController.ts:233-241`): only marks activity if the drag target is on the canvas — guards against UI button clicks triggering `isCameraInteracting` → resolution-scaling churn → accumulation reset.
- **Joystick channel** (`useInputController.ts:35-36, 169-176, 183-184`): subscribes to `window 'joyMove' / 'joyLook'` `CustomEvent`s carrying `{x, y}` `detail`; values feed into `joystickMove.current` / `joystickLook.current` consumed by `CameraController.update` in Fly mode.
- **`isInteracting()`** returns true if dragging, any move-key/joystick non-zero, or within 200 ms of the last activity timestamp (covers discrete scroll/key-tap events that don't keep `moveState` true) — `useInputController.ts:256-262`.
- **`usePhysicsProbe`** writes three HUD elements (`speed`, `dist`, `reset`) imperatively each frame, plus mirrors `engine.lastMeasuredDistance` from the depth readback — `usePhysicsProbe.ts:27-77`.
  - Speed HUD updates every 10th frame: `SPD {speedRef * 100}%` — `:27-31`.
  - Distance HUD shows `DST {formatDist(d)}` with color red `<1.0` else cyan; sky/invalid path uses gray `(sky)` label — `:33-38, 57`.
  - Reset button auto-show at `d > MAX_SKY_DISTANCE` OR `d < 0.001`, or always in sky branch — `:40-44, 58`.
- **Asymmetric distance smoothing**: large increases (`> prev * 1.5`) blend at 8 % per frame (~60 frames to converge); decreases snap instantly so speed drops immediately on surface approach — `usePhysicsProbe.ts:64-73`.
- **Physics probe modes** (`qualityState.physicsProbeMode`): 0 = auto depth-buffer readback, 1 = same auto path (no explicit branch — falls through to depth readback section), 2 = `manualDistance` override — `usePhysicsProbe.ts:93-104`.
- **Depth readback path** (`usePhysicsProbe.ts:106-167`): worker mode (`!renderer`) reads from `engine.lastMeasuredDistance` shadow state and pushes to `processDepthData`; direct mode samples a 3×3 neighbourhood around `(width/2, height/2)` via `engine.pipeline.readPixels` against `getPreviousRenderTarget()` (previous-frame to avoid GPU stall) and averages valid depth samples (`>0`, `<MAX_SKY_DISTANCE`, finite). Skips during initial shader compile (`!engine.hasCompiledShader || frameCount < 15`) — `:84-90`.
- **Focus Lock** (worker path only): when `store.focusLock` is on, syncs `optics.dofFocus` to smoothed distance if relative change `> 1%` and distance is in `(0, MAX_SKY_DISTANCE)` — `usePhysicsProbe.ts:114-123`.
- **HUD subscription model** (`HudOverlay.tsx:38-50`): self-subscribes to individual store fields (`tabSwitchCount`, `cameraMode`, `navigation`, `showHints`, plus three action callbacks) rather than taking parent props — limits HUD re-renders to actual visible changes. Synthetic `state`/`actions` objects are constructed locally for the existing prop-style consumer code.
- **HUD visibility logic**: subscribes to `useAnimationStore.isCameraInteracting` via `useAnimationStore.subscribe` (not the React selector), fades crosshair container at 2 s and bottom cluster at 10 s after interaction stops; both snap to opacity 1 immediately on interaction — `HudOverlay.tsx:61-94`.
- **HUD speed slider** (Fly only) maps drag x ∈ [0,1] across slider rect to `flySpeed = 10^(3x − 3)`, i.e., `[10⁻³, 10⁰] = [0.001, 1.0]` — `HudOverlay.tsx:106-129`. Haptic vibrate(5) at limits. Visual progress bar back-computes `(log10(flySpeed) + 3) / 3` — `:131-134`.
- **HUD region prop** (`'top' | 'bottom' | 'all'`, default `'all'`): the app splits crosshair-layer from bottom-pill cluster so the bottom cluster can be anchored to the viewport area outside the scaled canvas in Fixed mode — `HudOverlay.tsx:24-29, 136-138`.
- **Cursor-anchor toggle button** in HUD (Orbit only, non-mobile) toggles `state.navigation.orbitCursorAnchor`, with crosshair-glyph SVG; tooltip swaps between "around cursor" and "around centre" — `HudOverlay.tsx:224-256`.
- **Tutorial-hint stubs**: HUD imports `useTutorAnchor`, `mergeRefs`, `actionBus` from `engine/plugins/Tutorial` and wires anchors on the speed slider and reset button; full `<HintDisplay>` overlay is not ported (placeholder comment at `HudOverlay.tsx:259-261`).
- **`modifiers.ts`** is shared by both `useInputController` (wheel handler) and `Navigation` (wheel, middle-drag, custom orbit drag, drei zoom/rotate/pan speed assignments at `Navigation.tsx:1288-1291`). UI sliders use a separate `10×/0.1×` scale via `useDragValue.ts` — comment `modifiers.ts:5-6`.

## Invariants and gotchas

- **Camera resting state must be `position = (0,0,0)`** with world coordinates in `sceneOffset`. The 100 ms stop-debounce explicitly normalises this and bakes `camera.position` into `sceneOffset.{xL,yL,zL}` then `VirtualSpace.normalize` — `Navigation.tsx:1202-1216`.
- **Wheel + middle-drag absorb per-event, not at burst end.** The `scrollEndTimeout` is state-cleanup only; renaming it `flush` would be a misnomer per the explicit comment at `Navigation.tsx:327-339`.
- **`hoverPivotWorldRef` is NOT cleared on absorb** — clearing it would force a "fly to centre" when the cursor hadn't moved (10 px gate suppresses re-pick). `absorbGenRef` is the correct invalidation mechanism — `Navigation.tsx:274-282`.
- **Hover pre-pick skips during `isCustomOrbitRef`**, not just during `isOrbitDragging` / `isScrollingRef`. Missing this guard would risk a stale `absorbGenRef` miss mid-custom-orbit — `Navigation.tsx:531-536`.
- **Drei's `OrbitControls` runs at `useFrame` priority −1**, before Navigation's priority-0 frame. To suppress its `lookAt(target)` during a custom gesture you must flip `orbitRef.current.enabled = false` synchronously inside the pointer handler — flipping a ref alone lets one frame slip — `Navigation.tsx:722-726`.
- **Drei must be ENABLED for wheel-only PAN when cursor-anchor is OFF**, otherwise `enabled=false` at wheel-fire time silently no-ops zoom (no pointerdown precedes wheel-only interaction) — `Navigation.tsx:1273-1285`.
- **Q/E roll skips the manual `orbitRef.update()` when drei is enabled** to avoid double-multiplying `_sphericalDelta` by `(1 - dampingFactor)` — visible as a halved pointer-driven motion stagger — `Navigation.tsx:1304-1321`.
- **`MAX_SKY_DISTANCE` (imported from `../../data/constants`)** is the sky sentinel both for physics-probe acceptance and for the "show reset button" trigger — `usePhysicsProbe.ts:42, 48`. The integrator also guards `MISS_DIST = 1000.0` literal when writing `targetDistance` to store at `Navigation.tsx:1200`.
- **Distance smoothing is asymmetric** — instant snap on decrease is intentional for safety/responsiveness (prevents speed lag when diving toward a surface) — `usePhysicsProbe.ts:64-73`.
- **`engine.dirty` must be set inside per-event absorb paths** (wheel `:640`, custom orbit `:805`, middle-drag `:893`, pan per-frame `:1336`) because per-event absorb returns `camera.position` to zero each event — the integrator's `posChanged` check can't see the motion otherwise.
- **Roll velocity has two distinct refs** — `inputRollVel` is the input-controller's accumulator consumed by Navigation; the local `rollVelocity` ref at `Navigation.tsx:233` is declared but the integrator reads `inputRollVel` (renamed from destructure at `:248`).
- **OrbitControls.PAN distance** is computed from `camera.position.distanceTo(target)`. After a custom-orbit gesture leaves `target` at `camera.position + forward × 0.0001`, drei pan would be a no-op; `onStart` seats `target` ahead at `distAverageRef` distance to fix this — `Navigation.tsx:1373-1393`.
- **Wheel handler uses `passive: false`** so it can call `e.preventDefault()` — required for the custom translation math to deny the page from scrolling — `Navigation.tsx:589, 652`.
- **`incrementTabSwitchCount` is fired from a `useFractalStore.subscribe` on `s.cameraMode`** (not via the React `useEffect` deps mechanism) — this is in the HUD, not Navigation — `HudOverlay.tsx:97-104`. The HUD also gates the "Press Tab" hint behind `tabSwitchCount < 2` — `:264`.
- **`useInputController` ignores keys while `isTimelineHovered`** to keep timeline shortcuts un-shadowed by WASD — `useInputController.ts:78-79`.
- **Camera key capture (`captureCameraKeyFrame`) runs during `isRecording && recordCamera`** with `skipSnapshot: true` and interpolation `'Linear'` (when playing) or `'Bezier'` (when not) — `Navigation.tsx:1181-1186`.
- **Pointer-events-auto class is the catch-all UI gate** for nearly every Orbit-mode handler (`Navigation.tsx:529, 584, 687, 844, 952`). Adding new HUD widgets without that class will leak pointer events into the camera handlers.
- **Worker-mode physics probe** doesn't read pixels — it consumes `engine.lastMeasuredDistance` set by the worker's own depth readback. Direct mode (`engine.renderer` truthy) does the 3×3 readback locally — `usePhysicsProbe.ts:107-127, 129-167`.
- **`engine.queueOffsetSync(absorbed)` plus direct `useFractalStore.setState({ sceneOffset })`** is the "silent" path — bypasses `setSceneOffset` to skip OFFSET_SET re-emit and accumulation reset, while still keeping `getPreset()`/share-links accurate — `Navigation.tsx:311-323`.
- **`engine.cameraInUse` is set per-frame from Navigation** but persistent worker-side `cameraInUse` is driven by `GmtRendererTickDriver` (see comment `Navigation.tsx:1156-1162`). This is a `MARK_INTERACTION` pulse for the adaptive-resolution kick-in only.

## Open questions

- **orphan-sweep: `unifiedOffsetRef`** (`Navigation.tsx:173, 1340-1348`) is maintained every frame but I don't see any read inside this subsystem — likely consumed elsewhere (CameraController, snapshot pipeline?). Worth grep-confirming whether the shadow ref is still needed or whether all consumers now read `engine.sceneOffset` + `camera.position` directly.
- **orphan-sweep: `lastRot` rotation buffer** declared at `Navigation.tsx:230` is updated alongside `lastPos` but the `rotChanged` test only uses `lastRot.current.angleTo(...)` for the integrator's own deltas. Could be replaced with quaternion components if `THREE.Quaternion.angleTo` ever becomes a hotspot — currently fine, just noting.
- **orphan-sweep: Navigation-local `rollVelocity` ref** at `Navigation.tsx:233` is declared but the integrator only uses `inputRollVel` (`useInputController` destructure at `:248`). The local ref appears unread within this file — candidate for deletion if no external consumer.
- **orphan-sweep: `setIsCameraInteracting` selector** at `Navigation.tsx:163` is always truthy after subscription, yet `:1081, 1154` guard `if (setIsCameraInteracting)`. The guards look defensive against a possible undefined return from the store selector; harmless but could be tightened.
- **Pivot reticle DOM mutation** uses raw `document.createElement` / `appendChild` on `gl.domElement.parentElement` (`Navigation.tsx:920-942`). Strange-shaped relative to the React-rendered HUD reticle in `HudOverlay`; worth a follow-up whether unifying them under React-managed DOM is desirable, or whether the imperative path is intentional for performance.
- **`processDepthData` smoothing constants** (`prev * 1.5` threshold, `0.08` blend) are inline magic numbers — could move to `data/constants` alongside `MAX_SKY_DISTANCE` if any tuning surface is added later.
- **`region` prop split** in HUD (`'top'/'bottom'/'all'`) is documented in the prop comment (`HudOverlay.tsx:24-29`) but the rationale ("Fixed mode canvas-scale decoupling") lives only there — worth surfacing in subsystem doc when written.
- **Hint system stubs** (`ActiveHint = unknown`, NotPorted comment at `HudOverlay.tsx:259-261`): is the intent to port `HintDisplay` into engine-gmt, or to let apps layer their own? The current shape supports both but doc should pick one.
- **`isCameraLocked` predicate** (`Navigation.tsx:978`) duplicates similar logic likely present in animation-store consumers. Worth checking against the e03-animation survey for drift.
