## Public API surface

- `FluidToyApp` React component, default-mounted from `main.tsx:284-288` — the only top-level export consumed outside this folder.
- `appEngine`, `brushHandles`, `cursorHandles` typed singletons (`engineHandles.ts:27,37,52`) — the cross-tree IO points; anyone outside fluid-toy needing the live `FluidEngine` reads `appEngine.ref.current`.
- `installFluidToyViewLibrary()` (`viewLibrary.ts:221`) — single-call install of the saved-views state-library binding (consumed only from `main.tsx:264`).
- `setupFluidToy()` (`setup.ts:18`) — applies the panel manifest + un-collapses left dock; called once from `main.tsx:278`.
- `registerFluidToyHotkeys(engineRef)` (`hotkeys.ts:25`) — registers Space/R/O/Home; called from `useFluidEngine.ts:113` after engine boots.
- `runVideoExport` (`components/RenderDialog/exportRunner.ts`) — `RenderDialogRunner` impl passed to `installRenderDialog` at `main.tsx:276`.
- `FluidBucketController` class (`bucket/FluidBucketController.ts:1`) — implements `BucketRenderController`; instance handed to `installBucketRender` at `main.tsx:117-122`.
- Per-feature `<X>Feature: FeatureDefinition` and `sync<X>ToEngine(engine, slice, …)` exports from each `features/*.ts` — consumed by `registerFeatures.ts` and `useEngineSync.ts` respectively.
- Type augmentations in `storeTypes.ts:42-72` extend engine's `AppFeatureSlices` and `FeatureStateMap` — the side-effect that turns `useSlice('julia')` and `setJulia` into typed calls everywhere in this app.
- `viewLibrary.ts:39-57` augments `EngineStoreState` + `EngineActions` with `savedViews*`, `addView`, `selectView`, etc.

## Architecture

- Boot order is strict at `main.tsx`: side-effect feature registrations run first (`main.tsx:21` imports `./registerFeatures`), then migrations (`main.tsx:25`), then plugin installs, then `setupFluidToy()` at `main.tsx:278` and finally `ReactDOM.createRoot().render(<FluidToyApp/>)` at `main.tsx:284-288`. README's "registries freeze on first store touch" invariant is the reason for the ordering.
- `FluidToyApp.tsx:45-215` is intentionally thin: subscribes via granular Zustand selectors only, delegates all engine plumbing to three hooks — `useFluidEngine` (`:62`), `useEngineSync` (`:98`), `useDeepZoomOrbit` (`:99`). The header at `:46-51` explains why no `useEngineStore()` no-arg call is allowed (max-depth re-render guard).
- The render-size pipeline lives in one effect at `FluidToyApp.tsx:130-142`: `base (CSS px) × renderScale × quality → engine.setRenderSize`; the canvas drawing buffer AND the sim grid share that one resolution (commented at `:73-81`).
- TSAA pause/cap collapse onto `tsaaSampleCap` and `paused` engine knobs at `FluidToyApp.tsx:110-118`; K-sample-per-frame is fixed at 1 per `:120-126`.
- `useFluidEngine.ts:30-122` owns the RAF loop. Per-frame work: `stepBrush(...)` (`:65`), deterministic-playback override (`:82-93`, drives engine clock from `currentFrame * 1000/fps`), `engine.frame(engineT)` (`:94`), and 10 Hz `reportAccumulation` throttle (`:95-102`).
- `useEngineSync.ts:25-69` is the only file that subscribes to every DDFS slice. Slice→sync wiring uses one `useEffect` per slice; `applyLiveMod` (`:46-51`) folds `liveModulations` in via `useMemo` so syncs only re-fire on real change.
- Critical comment at `useEngineSync.ts:53-57`: `syncJuliaToEngine` deliberately omits `liveMod` from its deps so per-frame modulation ticks can't clobber gesture-set `engine.params.center/zoom` mid-pan/zoom; a separate effect at `:62` writes only `juliaC` from liveMod.
- `useDeepZoomOrbit.ts:32-139` rebuilds the reference orbit whenever julia view/iter or deepZoom flags change. LA/AT are Mandelbrot-only and power-2-only per gating at `:80-82`. Cancel-via-closure at `:84` + cleanup at `:135` handles superseded builds.
- `engineHandles.ts:27,37,52` are the three cross-tree handles (engine ref, brush runtime + cached LUT, cursor state); shape replaces a former grab-bag object per the file header.
- Slice migration v1 (`migrations.ts`) handles the 2026-04-23 tab-parity restructure: `dye→palette` rename plus per-field moves to `collision`/`composite`/`fluidSim`/`coupling`; `orbit` and `sceneCamera` slices removed (their fields folded onto `coupling` and `julia` respectively).
- `panels.ts:23-135` declares the dock layout — left dock (View / hidden Fractal / Deep Zoom / Palette / Modulation / Presets) vs right dock (Coupling / Fluid / Collision / Brush / Post-FX / Composite). Multi-section tabs use `items: [{type:'section', label}, {type:'feature', whitelistParams}]` to subdivide a single feature's params into visual groups.
- `viewLibrary.ts:81-128` implements a smooth 500 ms tween between saved views — log-space lerp on `zoom`, linear lerp on `center`/`juliaC`/`power`, `kind` and `maxIter` snap immediately. `DEFAULT_VIEWS` at `:186-197` seeds first-boot library.
- Pointer pipeline: `FluidPointerLayer.tsx:37-46` is a 0-DOM React shell that mounts `useModifierKeys` + `useCanvasContextMenu` + `useCanvasPointerHandlers`; the real dispatcher is `pointer/handlers.ts:55-71` which switches on `e.button` + `mods.b/c` to one of `splat / pan / zoom / pickC / resizeBrush` per file under `pointer/gestures/`.
- Store-bypass during pan/zoom is enforced in `pointer/handlers.ts:87-103`: gesture writes `engine.setParams({center,zoom})` + stashes `pendingViewRef`; only on pointerup does it call `setJulia` once. Same pattern for wheel via `wheel.ts`'s 100 ms idle commit timer (per README:189-200).
- Brush module split: `brush/color.ts` (pure colour resolver), `brush/particles.ts` (Particle type, spawn/step), `brush/emitter.ts` (`BrushRuntime`, `stepBrush`, `emitStrokeSplat`, `emitPressSplat`) — re-exported via `brush/index.ts` EXCEPT `readBrushParams` which is import-direct only (`brush/index.ts:48-58`).
- `FluidEngine.ts` (~1754 lines) is the WebGL2 orchestrator; owns four sub-controllers — `DeepZoomController`, `GpuTimerManager`, `GradientLutManager`, `BloomChain` (`FluidEngine.ts:21-25`). Public enums for cross-app use (`ForceMode`, `ForceSource`, `ShowMode`, `FractalKind`, `DyeBlend`, `DyeDecayMode`, `ToneMapping`, `ColorMapping`) all originate at the top of this file (`:28-117`).
- Presets pipeline: `presets/data.ts` carries STRING enums (legacy reference toy format), `presets/apply.ts:36-51` rebuilds each slice from its feature's defaults and merges the preset patch on top, dispatching one `set<Slice>` per slice. Avoids field leakage between sequential preset applies.
- Bucket render: `FluidBucketController.ts` uses two nested loops (image tiles → GPU sub-buckets), holds the canvas at full tile size while sub-buckets use `uRegionMin/Max` shader discard for consistent UV mapping. RGBA8 stitch (no Float32 HDR — no bloom inside per-tile composite).
- Video export: `exportRunner.ts` is a deterministic per-frame ratchet — scrub, apply mods, yield to React, force-pause sim and converge TSAA via `engine.frame()` spam, then advance sim clock by exactly `1/fps` and encode.

## Invariants and gotchas

- `brush/index.ts` MUST NOT re-export `readBrushParams` — `readBrushParams` pulls `useEngineStore`, and several feature files transitively reach the brush barrel via `engineHandles` during registration; importing the store before registry registration completes freezes the registry. Documented inline at `brush/index.ts:48-58`.
- `presets/apply.ts:17-23` deliberately avoids eager `useEngineStore` import for the same boot-order reason — it reaches the store via a global `__store` handle at call time.
- `useEngineSync.ts:58` excludes `liveMod` from the julia-sync effect's deps; without this, per-frame modulation ticks would clobber gesture-set engine params. Critical for pan/zoom not to snap-back during LFO-driven juliaC sweeps.
- Pan / middle-drag / wheel bypass the store during the gesture (`pointer/handlers.ts:87-103` + per-gesture files); single `setJulia` commit on pointerup or wheel idle. Reason: per-move `setJulia` cascades enough re-renders to trip React 18's max-depth guard (README `:189-200`).
- `useEngineStore()` no-arg subscription is forbidden in `FluidToyApp` — header at `FluidToyApp.tsx:46-51` explains; every read must be a granular selector.
- DD-precision pan: `julia.centerLow_*` carries the sub-f64 lo-word of a Dekker double-double accumulator and is registered hidden in camera tracks at `main.tsx:148-149`. Skipping it loses keyframes to f64 ulp quantization for zoom < 1e-15.
- Camera-pair binding at `main.tsx:166-170` makes interpolated pan velocity inherit zoom-track easing (`c(f) = c0 + (c1-c0)·(z(f)-z0)/(z1-z0)`) — needed for visually-uniform deep-zoom flythroughs.
- `installCamera({hideShortcuts:true})` at `main.tsx:202` — the camera plugin's slot keys are suppressed because `installFluidToyViewLibrary()` at `:264` registers the same Ctrl+1..9 / 1..9 bindings against `savedViews` instead.
- TSAA sample cap is overridden from default 256 to 64 at `main.tsx:78` (fluid-toy's TSAA needs less than GMT's path tracer; deep-accum gate at sampleCap/2 = 32 stops adaptive re-engaging too eagerly).
