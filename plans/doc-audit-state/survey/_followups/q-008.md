## Answer

Both exports are **Zustand-selector-backed** — there is no React context provider. The coupling is via the shared `useEngineStore` (`store/engineStore`), which the e12 audit already documented; this followup confirms the topbar's specific usage pattern.

### `useMobileLayout` (reactive hook)
`hooks/useMobileLayout.ts:66-71` — subscribes to three engine-store fields via Zustand selectors:

```
const pref = useEngineStore((s) => s.uiModePreference);
const isDeviceMobile = useEngineStore((s) => s.isDeviceMobile);
const isPortrait = useEngineStore((s) => s.isPortrait);
return { isPortrait, isDeviceMobile, isMobile: resolveIsMobile(pref, isDeviceMobile) };
```

Returns three flags so each call site can pick the right gating policy (preference-aware `isMobile` vs raw `isDeviceMobile`).

### `isMobileSnapshot` (non-reactive predicate)
`hooks/useMobileLayout.ts:45-48` — reads from the same store via `useEngineStore.getState()`:

```
const { uiModePreference, isDeviceMobile } = useEngineStore.getState();
return resolveIsMobile(uiModePreference, isDeviceMobile);
```

Used in menu/topbar `when:` predicates where hooks are unavailable. Does not subscribe — safe outside renders, unsafe inside them.

### How the store stays in sync with the window
Importing `hooks/useMobileLayout.ts` installs a module-level singleton `window.resize` listener (`useMobileLayout.ts:25-37`) that writes `{ isDeviceMobile, isPortrait }` into `useEngineStore` whenever the breakpoint or orientation flips. Initial values are seeded by the slice initializer at `store/slices/uiSlice.ts:180-183`. `uiModePreference` is persisted to `localStorage` under `'gmt.uiModePreference'` (`store/slices/uiSlice.ts:11`).

### topbar.tsx coupling (the question's specific case)
`engine-gmt/topbar.tsx:37` imports both symbols from `../hooks/useMobileLayout`. Two distinct patterns:

1. **Predicate form** for menu `when:` callbacks (called per re-evaluation, no hook context):
   - `:47` — `const desktopOnly = () => !isMobileSnapshot();`
   - `:395`, `:400`, `:408`, `:427` — `when: () => isMobileSnapshot()` gates the Quality section, MobileQualityMenuItem, "Reduce Quality on Touch", and the display separator.
   - Comment at `:45-47` notes TopBarHost re-evaluates predicates on every store update affecting the relevant fields, i.e. reactivity comes from TopBarHost's own subscription, not from `isMobileSnapshot` itself.

2. **Hook form** inside a React subcomponent:
   - `:245-247` — `CenterHUDWrapper` calls `const { isMobile } = useMobileLayout();` to derive the `isMobileMode` prop for `CenterHUD`.

### Conclusion
No context provider; no parallel state. Both APIs are thin wrappers over `useEngineStore` (selectors for the hook, `getState()` for the snapshot). The "sync with global engine store" is the entire coupling mechanism — there is nothing else. The asymmetric usage in topbar (`isMobileSnapshot` for menu predicates, `useMobileLayout` inside the React tree) is the documented pattern, not a drift.
