## Question
> Camera Manager panel (engine-gmt/panels.ts:464-470) has `dock: 'left', order: 20, component: 'panel-cameramanager'` but topbar.tsx has no explicit registration. Is it registered elsewhere? Comment says "Opens as a floating panel on demand via Camera menu" but dock is 'left' not 'float'.

## Answer
The `panel-cameramanager` component **is** registered — by the engine-gmt UI feature module at `engine-gmt/features/ui.tsx:175`:

```
componentRegistry.register('panel-cameramanager', CameraManagerPanel);
```

This sits alongside the other bespoke panel registrations (`panel-engine`, `panel-graph`) in the same module (`features/ui.tsx:173-176`). The `componentRegistry` lookup is what `panels.ts` references via the manifest's `component:` field. So registration is fine; the topbar is **not** responsible for component registration — `topbar.tsx` only owns menu actions (`openCameraManager` callback at `topbar.tsx:185-198`), which call `togglePanel('Camera Manager', true)` from the host app.

The stale-comment observation is the real finding. `panels.ts:453-458` describes a previous configuration (`dock: 'float'` + `isCore: false` + topbar-only activation), but `panels.ts:459-470` — the actual current entry — uses `dock: 'left'` with `label: 'View Manager'`. The two comment blocks were not reconciled when the panel was moved from float to the left dock. The current behavior: the panel **does** live in the left dock (which is "hidden-by-default" per the second comment — the left dock surfaces when activated via `Dock.tsx`), and the topbar Camera menu's "View Manager" item (`topbar.tsx:316`) opens it via `togglePanel`. The `cameraSlice.ts:255` reference to `panelId: 'Camera Manager'` confirms the canonical PanelId is unchanged even though the user-visible label is "View Manager".

The default callback at `topbar.tsx:198` (`console.info('[gmt] Camera Manager panel not registered yet')`) is a fallback if the host app forgot to wire `openCameraManager`; it does not imply the component isn't registered — it implies the open action wasn't supplied. In dev/app-gmt this callback is overridden to call `togglePanel`.

## Confidence
high — the registration is grep-confirmed at `features/ui.tsx:175`, and the dock/label divergence in panels.ts is visible in a single read. The "registered elsewhere" question has a definitive answer.
