### Note on `componentRegistry`
The `componentRegistry` referenced in q-068 is the *consumer-side* parameter to `validateComponentRefs(componentRegistry)` (engine/FeatureSystem.ts:615–649) — it's not a structure owned by FeatureSystem itself, it's a `Record<string, React.ComponentType>` that the host app passes in so FeatureSystem can verify that every `customUI.component` string in registered features resolves to a real component. The shape and contents of that map are owned by the app, not the engine, and are documented per-app in the respective app surveys.

### What e10's doc should do
Cite e01 by name for any FeatureSystem signature; do not duplicate the interface text. The `tabConfig`/`menuConfig`/`menuItems`/`viewportConfig` fields are all documented in e01's "Public API surface" with line citations.
