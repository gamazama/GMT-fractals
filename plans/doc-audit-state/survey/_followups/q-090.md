## Question
> `components/topbar/` is empty — was content moved elsewhere or never built?

## Answer
**Content moved.** `components/topbar/` is an empty leftover directory; all former contents were relocated during the engine extraction. Git history (`git log --diff-filter=D --summary -- components/topbar/`) shows the directory once held at least 10 components — `FpsCounter.tsx`, `HelpMenu.tsx`, `CameraTools.tsx`, `SystemMenu.tsx`, `AdaptiveResolution.tsx`, `BucketRenderControls.tsx`, `CenterHUD.tsx`, `RenderTools.tsx`, `ViewportQuality.tsx`, plus more — all deleted across the engine-extraction stages (commit `26bb0b6` "Stage 10: strip fractal-shaped UI components", `a74c30a` "Stage 13", `2d85139` "Stage 14", and finally `3caa953` "cleanup: delete 17 orphan files identified by knip" which removed the last straggler `FpsCounter.tsx`).

The content lives in two places now: generic engine plugins under `engine/plugins/topbar/` (`FpsCounter.tsx`, `ProjectName.tsx`, `PauseControls.tsx`, `BucketRenderController.ts`, `BucketRenderPanel.tsx`, `installBucketRender.tsx`) and GMT-specific topbar widgets under `engine-gmt/topbar/` (`AdaptiveResolution.tsx`, `CenterHUD.tsx`, `ViewportQuality.tsx`, `Logo.tsx`, `ShareLinkButton.tsx`, `GmtBucketController.ts`). The top-level orchestrator is `engine/plugins/TopBar.tsx`. The empty `components/topbar/` directory should be deleted — it serves no purpose.

## Confidence
high — git history explicitly shows every file deletion under `components/topbar/`, and the destination files exist in `engine/plugins/topbar/` + `engine-gmt/topbar/` with matching names (e.g. `AdaptiveResolution.tsx`, `CenterHUD.tsx`, `ViewportQuality.tsx`, `FpsCounter.tsx`).
