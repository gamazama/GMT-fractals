## Answer

The GMT camera_manager subsystem is **split across coverage gaps and partial coverage**. None of the existing subsystem audits cover it end-to-end. Specifically:

### Coverage map of camera_manager subsystem files

| File | Lines | Covered by | Tier / depth |
|------|-------|------------|--------------|
| `engine-gmt/features/camera_manager/index.ts` | 16 | **g05-engine-gmt-features** (frontmatter line 77-80) | Tier C, fully read |
| `engine-gmt/features/camera_manager/CameraManagerPanel.tsx` | 154 | **NOT covered** — only mentioned by name at g05:178 ("Bespoke `CameraManagerPanel` component registered as `panel-cameramanager`"). Not in any survey's frontmatter. | None |
| `engine-gmt/features/camera_manager/logic.ts` | 128 | **NOT covered** — not mentioned in any survey. Houses `getDirectionName` (cardinal-axis label suggestion) imported by `cameraSlice.ts:33`. | None |
| `engine-gmt/store/cameraSlice.ts` | 310 | **NOT covered** — not in any g* slice's frontmatter. g05 references it by name only at lines 156, 178, 199 to note state ownership ("state owned by `cameraSlice`", "state lives in `cameraSlice` (not in the DDFS feature slice)"). | None |

### Why g04 does not cover it

g04 is `engine-gmt/navigation/*` (the flight-stick / orbit / fly-cam input layer), not the saved-camera library. Grep of g04-navigation.md for `cameraSlice|CameraManagerPanel|camera_manager` returns zero matches. `navigation/` (g04) and `camera_manager/` (this subsystem) are sibling but distinct subsystems.

### Why g05 only partially covers it

g05 surveyed `engine-gmt/features/*` and intentionally documented `CameraManagerFeature` as a "pure tab registration" (g05:156, g05:199) — its DDFS slot is empty. The survey then *acknowledges* the real implementation lives in `cameraSlice` (g05:178, g05:199) but does not audit that file because `engine-gmt/store/` was out of scope for g05 (which targeted `engine-gmt/features/`). The same g05 open-questions list flags `engine-gmt/store/engineStore` as an orphan-sweep candidate (g05:217), confirming that the engine-gmt store layer is a known gap.

### Why e09 only partially covers it

e09-camera-plugin audited the **engine-core** side: `engine/plugins/Camera.ts` (slot plugin), `engine/appHandles.ts`, `engine/migrations.ts`, `engine/store/createStateLibrarySlice.ts`, `engine/store/installStateLibrary.ts`. It documents the factory contract (`installStateLibrary<T>`) but not GMT's *consumer* of that contract, which is what `cameraSlice.ts:254` is doing.

### What cameraSlice.ts actually contains (310 lines)

Per the file header (`cameraSlice.ts:1-23`) and the `installStateLibrary` call site (`cameraSlice.ts:254-291`):

- **GMT-specific capture/apply** for split-precision `sceneOffset`, camera rotation quaternion, `targetDistance` (raymarched surface probe), and `OpticsState`.
- `SavedCameraPayload = CameraState & { optics: OpticsState }` (`cameraSlice.ts:41-43`) — the `T` for `installStateLibrary<SavedCameraPayload>`.
- `captureCameraState` reads via `CameraUtils` + `WorkerProxy`; `applyCameraState` writes via Zustand `setState` + `FractalEvents.CAMERA_TRANSITION` + `resetAccumulation`.
- `resetToFormulaDefault` walks the active formula's default preset.
- `suggestCameraLabel` delegates to `getDirectionName` in `features/camera_manager/logic.ts` for cardinal-axis labels ("Front View", "Top View").
- `captureCameraThumbnail` calls `engine.captureSnapshot()` on the worker.
- `isCameraModified` dirty-checks against current live state for the modified-marker.
- Calls `installStateLibrary<SavedCameraPayload>` with `panelId: 'Camera Manager'`, `arrayKey: 'savedCameras'`, `activeIdKey: 'activeCameraId'`, all 8 action names, slot-shortcut category `'Camera'`, and explicit `menu: null` (GMT wires its Camera menu by hand in `engine-gmt/topbar.tsx`).
- Header comment at `cameraSlice.ts:20-22` documents a load-order invariant: **`CameraManagerPanel` crashes on `savedCameras.length` until `installGmtCameraSlice()` runs**. `app-gmt/main.tsx` calls it right after `registerGmtUi()`.

### Verdict

**FAIL** (coverage gap, not a question that can be fully answered without a new survey slice). The camera_manager subsystem needs its own audit slice — recommend naming it something like **g11-gmt-camera-manager** covering:

- `engine-gmt/store/cameraSlice.ts` (310 lines) — the `installStateLibrary` consumer.
- `engine-gmt/features/camera_manager/CameraManagerPanel.tsx` (154 lines) — the bespoke panel, currently registered as `panel-cameramanager` in g05 with no source-level audit.
- `engine-gmt/features/camera_manager/logic.ts` (128 lines) — `getDirectionName` plus any other helpers.
- `engine-gmt/features/camera_manager/index.ts` (16 lines) — already covered by g05 but should be re-listed for completeness.

Adjacent invariants this new slice should call out:
1. **Load-order invariant** — `installGmtCameraSlice()` must run before any render of `CameraManagerPanel` (`cameraSlice.ts:20-22`).
2. **Boundary with e09** — `installStateLibrary<SavedCameraPayload>` is the contract handoff; the GMT side owns capture/apply/thumbnail/isModified/suggestLabel/onReset, the engine-core side owns the snapshot array + active id + bookkeeping actions.
3. **Boundary with g05** — `CameraManagerFeature` (DDFS) deliberately has empty params; the real state lives in `cameraSlice` (g05:199). The new slice should reciprocally call out that the DDFS feature is intentionally a tab-only stub.
4. **Boundary with engine-gmt/topbar.tsx** — `menu: null` in the `installStateLibrary` call (`cameraSlice.ts:254`) opts out of the bundle's auto-generated topbar menu because GMT's Camera menu (Save/Recall slots + Undo Move / Redo Move / Reset Position / View Manager) is wired by hand in topbar.tsx. The Camera menu items and the slot hotkeys both route to the same library actions, so menu clicks and hotkeys agree by construction.

## Report

FAIL — coverage gap confirmed. `engine-gmt/store/cameraSlice.ts` (310 lines), `engine-gmt/features/camera_manager/CameraManagerPanel.tsx` (154 lines), and `engine-gmt/features/camera_manager/logic.ts` (128 lines) are not audited by any existing g* slice. g04-navigation does not touch them. g05-engine-gmt-features only audits the 16-line `camera_manager/index.ts` (pure tab registration) and references the panel + slice by name only. e09-camera-plugin audits the engine-core factory (`installStateLibrary`) but not GMT's consumer of it. Recommend a new slice (suggested id: g11-gmt-camera-manager) covering the four files above.
