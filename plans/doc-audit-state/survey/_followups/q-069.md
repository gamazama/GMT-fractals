## Question
> `useEngineStore` shape, `setLiveModulations`, `lfosEnabled`, default DDFS setter generation, `liveModulations` slice — referenced from applyAt.ts:16, 29 and ModulationEngine.ts but defined in `store/engineStore` outside scope.

## Findings

### `useEngineStore` shape (composition)

`store/engineStore.ts` builds a single Zustand store typed as `EngineStoreState & EngineActions` via `storeFactory` (lines 85–401). The shape is composed of:

**Slices** (spread into the root state, lines 91–96):
- `createUISlice` (`./slices/uiSlice`)
- `createRenderControlSlice` (`./slices/renderControlSlice`)
- `createViewportSlice` (`./slices/viewportSlice`)
- `createScalabilitySlice` (`./slices/scalabilitySlice`)
- `createHistorySlice` (`./slices/historySlice`)
- `createFeatureSlice` (`./createFeatureSlice`) — generates the DDFS feature slices

**Engine-level scalar state** (lines 98–106):
- `formula: ''`
- `projectSettings: { name, version, author }`
- `lastSavedHash: null`
- `animations: []`
- `lfosEnabled: true`
- `liveModulations: {}`

**Actions** on the root: `setFormula`, `setProjectSettings`, `prepareExport`, `setAnimations`, `setLfosEnabled`, `setLiveModulations`, `loadPreset`, `loadScene`, `getPreset`, `getShareString`.

### `lfosEnabled`

A **top-level scalar** on the root store, not a slice. Default `true` (line 104). Setter `setLfosEnabled: (v) => set({ lfosEnabled: v })` (line 247). Serialized into presets at line 379 (`p.lfosEnabled = s.lfosEnabled`). Consumed by `applyAt.ts:21` to gate oscillator + rule updates.

### `liveModulations`

A **top-level scalar** on the root store (line 105), default `{}`. NOT defined in a separate slice — it lives directly on the engine store root alongside `animations` and `lfosEnabled`. Setter `setLiveModulations: (v) => set({ liveModulations: v })` (line 249). Written from `applyAt.ts:29` with a clone of `modulationEngine.offsets`.

### `setLiveModulations`

Defined at `store/engineStore.ts:249`. One-liner: `(v) => set({ liveModulations: v })`. No middleware, no derived recomputation — purely a setter that replaces the offsets map. Referenced from `applyAt.ts:29` to publish modulation offsets each frame.

### Default DDFS setter generation

DDFS feature slices are NOT generated inline in `engineStore.ts`. They are produced by `createFeatureSlice(set, get, api)` (imported from `./createFeatureSlice`, spread at line 96). The engine-store root carries only formula-level scalars and engine-wide actions; per-feature state and setters are owned by `createFeatureSlice`. This file deliberately does not enumerate feature setters — investigation of generation logic itself is out of scope for q-069 and lives in `store/createFeatureSlice.ts`.

### Lazy store instantiation

The hook (line 427) and global handles (lines 434, 442) wrap `_store` behind `ensureStore()` (lines 414–417). Rationale per comment block (lines 77–84, 403–411): production rollup chunking may evaluate this module before app-side `registerFeatures.ts` runs; deferring `create()` to first access keeps the feature registry unfrozen until app-side registration completes. The hook surface (`useEngineStore(selector, equalityFn)`) plus static `getState` / `setState` / `subscribe` all route through `ensureStore`.

### Related cross-store accessor

`getShaderConfigFromState(state)` (lines 480–506) builds a flat config snapshot that includes `state.formula`, render-control fields, every registered feature slice, and GMT-Modular top-level fields (`pipeline`, `graph`, `pipelineRevision`). Used by `loadScene` (line 327) to flush a full config to the worker.
