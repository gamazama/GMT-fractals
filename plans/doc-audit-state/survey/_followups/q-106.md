## Question
> The 5 legacy aliases (UberMenger, FoldingBrot, HyperTorus, HyperbolicMandelbrot, RhombicIcosahedron) — confirm whether saved scenes in the wild still reference these. If not, they're candidates for removal in a future major version.

## Answer
**In-tree half (high confidence):** No bundled scenes reference any of the five legacy alias names. The five aliases are registered at `engine-gmt/formulas/index.ts:107-111` (`UberMenger -> MengerAdvanced`, `FoldingBrot -> BoxBulb`, `HyperTorus -> Mandelorus`, `HyperbolicMandelbrot -> MandelBolic`, `RhombicIcosahedron -> Coxeter`). A repo-wide grep finds zero hits inside `public/gmf/**` (8 bundled `.gmf` scenes) and zero hits inside `public/formulas/*.json` (manifest, dec, v3-v4-catalog, gallery). The only non-source references are doc/plan files and two human-readable description strings (`MengerAdvanced.ts:8` "Formerly UberMenger", `BoxBulb.ts:8` "Formerly FoldingBrot") — these are descriptive text, not identifiers, and don't affect load. `engine-gmt/formulas/index.ts:53` carries a `// Formerly HyperTorus` comment on the `Mandelorus` import, also non-binding. Conclusion: the in-tree case is safe to remove the alias registrations whenever desired.

**Wild scenes half (unanswerable from code):** Whether users have `.gmf` or PNG-embedded-GMF files on disk (or in third-party galleries) that still serialise `formula: "UberMenger"` etc. cannot be determined from the repo. The aliases were renamed at unknown points pre-0.9.x, so any scene saved before that rename would still carry the legacy name. Recommended resolution path: (a) check the Supabase gallery DB (`project_gallery_supabase.md`) for `formula` column values matching the five legacy names — that covers the public gallery slice; (b) treat removal as a one-version deprecation — emit a `console.warn('legacy alias X -> Y, will be removed in 1.x')` from `registerAlias` resolution for one release, then drop in the major bump. Either is cheap; (a) gives a hard answer for the gallery subset, (b) covers private libraries we can't see.

## Confidence
**Partial — in-tree half high, wild half unanswerable.** The grep covers every bundled scene and every formula-metadata JSON in `public/`, so the in-tree verdict is concrete. The wild-scenes verdict is structurally unreachable from source code; it needs either a gallery DB scan or a deprecation cycle.
