## Question
> Whether downstream `app-gmt`, `demo`, `fluid-toy`, or future GMT-raymarching plugins ever call `defineFeature` instead of `: FeatureDefinition` — not visible from this subsystem alone.

## Answer
Workspace-wide grep for `defineFeature(` finds exactly **one** production call site: `demo/DemoFeature.ts:10` (`export const DemoFeature = defineFeature({ ... })`). Every other hit is documentation prose (`docs/engine/02_Feature_Registry.md`, `docs/engine/09_Bridges_and_Derived.md`, `docs/modules/engine/features.md`), source comments in `engine/features/setFeature.ts:5,9`, or audit-state notes. By contrast, `: FeatureDefinition` is the dominant in-code form across every app that ships features. Inspection of `engine/features/setFeature.ts:48-54` confirms `defineFeature` is a pure identity helper (`def => def`) whose only purpose is preserving the `params` record's literal types so `setFeature` / `getFeature` / `FeatureState<typeof X>` can infer per-key value shapes — it does no registration or mutation.

Per-app tally:

| App / package | `defineFeature(` call sites | `: FeatureDefinition` annotation files |
|---|---|---|
| `app-gmt/` | 0 | 0 (no feature defs live here — it imports from engine/engine-gmt) |
| `demo/` | 1 (`demo/DemoFeature.ts:10`) | 0 |
| `engine/` | 0 | 7 feature files + 1 in `FeatureSystem.ts` (8 files, ~13 occurrences) |
| `engine-gmt/` | 0 | 20 feature files |
| `fluid-toy/` | 0 | 10 feature files (+ 1 doc reference in `CODE_MAP.md`) |
| `fractal-toy/` | 0 | 3 files (`features/lighting.ts`, `features/camera.ts`, `renderer/formulaRegistry.ts`) |

Verdict: **adoption of `defineFeature` is effectively nil** — only the throwaway `demo` package uses it, and even the docs that promote the helper (`docs/modules/engine/features.md:87,146-158`) acknowledge the `: FeatureDefinition` annotation is what real code uses. The helper exists and works, but no GMT-raymarching plugin, no engine-gmt feature, no fluid-toy feature, and no app-gmt code calls it. The `: FeatureDefinition` form wins ~44-to-1 in shipped feature files.

## Confidence
high — `defineFeature(` is a uniquely greppable token, the helper is confirmed identity-only by direct read of `engine/features/setFeature.ts:48-54`, and the call-site inventory is exhaustive (every non-doc hit accounted for).
