# q-026: Where is `UniformDefinition.backingOnly` actually enforced?

## TL;DR

**PASS — enforcement is at shader-emit time in `shaders/chunks/uniforms.ts`,
not in `MaterialController` or a `UniformManager`.** The single check that
makes `backingOnly` mean anything lives at `shaders/chunks/uniforms.ts:8`
(and its F14-cloned twin `engine-gmt/shaders/chunks/uniforms.ts:8`):

```ts
UNIFORM_SCHEMA.forEach(u => {
    if (u.backingOnly) return; // Three.js backing only — GLSL declaration injected conditionally by features
    if (u.arraySize) {
        glsl += `uniform ${u.type} ${u.name}[${u.arraySize}];\n`;
    } else {
        glsl += `uniform ${u.type} ${u.name};\n`;
    }
});
```

The Three.js backing always exists because `createUniforms()` in
`engine/UniformSchema.ts:90-104` walks the schema **without** checking
`backingOnly` — every entry gets a `{ value }` slot. So the flag only
suppresses the static GLSL `uniform …;` declaration emitted into the
`UNIFORMS` chunk; it does nothing at JS-side uniform sync time.

The hint that this lives in `MaterialController` or `UniformManager` was a
red herring — those layers never read the flag. They sync whatever's in
`material.uniforms`, which includes the backing-only entry for parity.

## Evidence: only two enforcement sites

`Grep "backingOnly"` across the whole tree returns these production matches:

| File:Line | Role |
|-----------|------|
| `engine/UniformSchema.ts:21` | Type field on `UniformDefinition` interface (declaration) |
| `engine-gmt/engine/UniformSchema.ts:21` | F14 mirror of the interface |
| `engine-gmt/features/core_math.ts:131` | Single producer: `ModularParams` declared `backingOnly: true` |
| `shaders/chunks/uniforms.ts:8` | **Enforcement** — `if (u.backingOnly) return` skips GLSL emit |
| `engine-gmt/shaders/chunks/uniforms.ts:8` | F14 mirror of the enforcement |

That's the entire surface. No `MaterialController.ts`, no `UniformManager`,
no `WorkerProxy` references the flag. The only enforcement is at GLSL-string
generation time, in the chunk that builds the static `precision highp …;\n
uniform … ;` preamble for every shader.

## Evidence: `createUniforms()` ignores the flag

`engine/UniformSchema.ts:90-104`:

```ts
export const createUniforms = () => {
    const uniforms: Record<string, { value: any }> = {};
    
    UNIFORM_SCHEMA.forEach(def => {
        let val = def.default;
        if (val && typeof val === 'object') {
            if (val.clone) val = val.clone();
            else if (Array.isArray(val)) val = val.map((v: any) => v.clone ? v.clone() : v);
            else if (val instanceof Float32Array) val = new Float32Array(val);
        }
        uniforms[def.name] = { value: val };
    });
    
    return uniforms;
};
```

No `if (def.backingOnly)` branch. Every schema entry — including the
`Float32Array(MAX_MODULAR_PARAMS)` backing — lands in the returned
`uniforms` record. This is why `syncModularUniforms()` can write into
`uniforms[Uniforms.ModularParams].value` for every formula without first
checking which one is active: the backing always exists.

## Evidence: the conditional GLSL declaration is feature-injected, not auto-emitted

The `backingOnly` skip in the chunk only suppresses the **static** preamble
declaration. The actual `uniform float uModularParams[64];` line still has
to appear when `formula === 'Modular'`, and it does — via
`CoreMathFeature.inject()` at `engine-gmt/features/core_math.ts:152-161`:

```ts
inject: (builder, config) => {
    const formula = config.formula as FormulaType;
    // ...
    // 1. Modular pipeline revision (forces recompile when graph changes)
    if (formula === 'Modular') {
        builder.addDefine('PIPELINE_REV', (config.pipelineRevision || 0).toString());
        // Declare uModularParams only for Modular shaders (backing always exists via extraUniforms)
        builder.addUniform(Uniforms.ModularParams, 'float', MAX_MODULAR_PARAMS);
    }
    // ...
}
```

The contract is symmetric: feature *opts in* to the backing via
`backingOnly: true` (always present, never auto-declared), and feature
*opts in* to the GLSL declaration via `builder.addUniform(…)` inside its
own `inject()` (declared only when needed). The chunk's `if
(u.backingOnly) return` is the bridge — it stops the schema walker from
double-emitting what the feature is already emitting itself.

## Evidence: the CHANGELOG entry confirms the design

`docs/CHANGELOG_DEV.md:586`:

> **`uModularParams` GLSL conditional declaration**: Added `backingOnly:
> true` flag to `UniformDefinition`. Uniform still creates a Three.js
> backing for all formulas, but the GLSL `uniform float
> uModularParams[64]` declaration is now only emitted for `formula ===
> 'Modular'` shaders. Files: `engine/UniformSchema.ts`,
> `shaders/chunks/uniforms.ts`, `features/core_math.ts`.

The changelog names exactly the three files the grep found — no mention of
`MaterialController`, no mention of `UniformManager`. That's the whole
mechanism.

## Why the survey couldn't see it

The e04 survey covered `engine/ShaderBuilder.ts`, `engine/ShaderFactory.ts`,
and adjacent compose-chain files. The enforcement lives in
`shaders/chunks/uniforms.ts` — a sibling tree (`shaders/chunks/`, not
`engine/`) that the survey scoped out. Survey line 67 was correct that
"enforcement must live downstream" — it just happens to be downstream in
the chunk producer rather than in a runtime sync layer.

## Doc gap (minor)

`docs/gmt/03_Modular_System.md:41` already documents the flag and points
at `shaders/chunks/uniforms.ts` as the enforcement site, so the gap is
upstream: the `UniformDefinition` JSDoc comment at
`engine/UniformSchema.ts:19-21` ("creates the Three.js uniform backing
but skips GLSL declaration") doesn't say *where* the skip happens. A
one-line addition — "Skip is enforced in `shaders/chunks/uniforms.ts`;
features that need the declaration must call `builder.addUniform(…)` in
their `inject()`" — would close the loop for anyone reading the schema
file first.

## Verdict

**PASS.** `backingOnly` is enforced at exactly one site:
`shaders/chunks/uniforms.ts:8` (and its F14 mirror at
`engine-gmt/shaders/chunks/uniforms.ts:8`). The flag suppresses the
static GLSL `uniform` declaration in the preamble chunk while leaving
the Three.js uniform backing intact (via `createUniforms()` which
ignores the flag). Re-declaration when needed is the feature's
responsibility, done via `builder.addUniform(…)` in `inject()` — the
only current producer is `CoreMathFeature` for `uModularParams` when
`formula === 'Modular'`. No `MaterialController` or `UniformManager`
involvement; the original hint pointing at those layers was wrong but
the survey's framing ("enforcement must live downstream") was right.
