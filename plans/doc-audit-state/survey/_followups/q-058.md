## Question
> `lookup(key)` (Shortcuts.ts:164-167) and `clear()` (Shortcuts.ts:168-170) are on the public registry but no caller seen in audited files — usage probably in shortcut-help or test infra.

## Answer
**Verdict: dead code (in callers).** Workspace-wide grep finds zero runtime callers of either `shortcuts.lookup(` or `shortcuts.clear(`. The only `lookup` hits are unrelated (`binderRegistry.lookup(id)` in `engine/AnimationEngine.ts:82`, plus a comment in `engine/plugins/topbar/PauseControls.tsx:51`) and a single docs mention in `docs/engine/07_Shortcuts.md:182` that documents `shortcuts.lookup('Ctrl+Z')` as an introspection example but never actually invokes it. The `.clear(` hits in `debug/` (`test-dopesheet-renderer-cache.mts`, `test-graph-renderer-cache.mts`, etc.) are all on cache objects, not the shortcuts registry. No shortcut-help UI exists — the help overlay (`engine/plugins/Help.tsx`) iterates via `shortcuts.list()`, not `lookup()`. No test file imports or exercises these methods.

Both methods remain part of the public registry surface (`Shortcuts.ts:164-170`) and the doc advertises `lookup()` as an introspection helper, so they are best classified as **intentional public API kept for future debug/help tooling**, not callers-yet-to-be-found. The audit can safely note them as "exposed but currently unused" rather than chasing further consumers.

## Confidence
high — exhaustive grep across `app-gmt/`, `fluid-toy/`, `fractal-toy/`, `demo/`, `engine/`, `components/`, `debug/`, and root-level entrypoints turned up no caller. The Shortcuts source confirms the methods are only defined on the exported `shortcuts` registry and not consumed internally (the dispatcher uses `shortcuts.list()` at line 186, not `lookup`).
