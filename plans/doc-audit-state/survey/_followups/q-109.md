## Question
> `region` prop split in HUD (`'top'/'bottom'/'all'`) is documented in the prop comment (`HudOverlay.tsx:24-29`) but the rationale ("Fixed mode canvas-scale decoupling") lives only there — worth surfacing in subsystem doc when written.

## Answer
The `region` prop on `HudOverlay` lets the app render the crosshair/reticle layer separately from the nav pill cluster so the bottom cluster can be anchored to the viewport area outside the scaled canvas in Fixed-resolution mode (where the canvas itself is scaled and would otherwise shrink the bottom cluster with it). The full rationale is currently only present as a JSDoc block above the prop declaration at `engine-gmt/navigation/HudOverlay.tsx:24-29`, with the default `'all'` set at line 38. The g04-navigation survey (`plans/doc-audit-state/survey/g04-navigation.md:63`) summarises this in one bullet but doesn't include the verbatim wording, and lists it as an open question at line 103 expecting it to be surfaced when the subsystem doc is written.

Preservable: "Which slice of the HUD to render. 'top' = crosshair + reticle (canvas-relative, tied to view direction). 'bottom' = nav pill cluster + tab hint (viewport-area-relative — anchor outside the scaled canvas in Fixed mode). 'all' = both, default. App splits the two so the bottom cluster doesn't shrink with the canvas." — `engine-gmt/navigation/HudOverlay.tsx:24-29`.
