## Question
> Orphan-sweep candidate: dead profiling block. `ShaderBuilder.buildFragment` Main variant builds a `sections` array and computes `profile`/`totalSize` (ShaderBuilder.ts:577-601) but never reads or returns them. Either wire to a debug uniform/console, or delete. Prefix: `orphan-sweep: ShaderBuilder section-size profile dead-code`.

## Answer

Confirmed dead. `engine-gmt/engine/ShaderBuilder.ts:577-601` builds an 18-entry `sections` array, derives a `profile` string of per-section sizes in kb (`engine-gmt/engine/ShaderBuilder.ts:597-600`) and a `totalSize` sum (`engine-gmt/engine/ShaderBuilder.ts:601`), then immediately falls through to the `return` template literal at `engine-gmt/engine/ShaderBuilder.ts:603-632` which interpolates the raw section strings directly and never references `sections`, `profile`, or `totalSize`. A repo-wide grep for these identifiers in the file returned only the three definition lines — no `console.log`, no debug uniform, no exported getter consumes them.

Note this is specific to the GMT-only `engine-gmt/engine/ShaderBuilder.ts` (634 lines). The generic engine fork at `engine/ShaderBuilder.ts` (136 lines) is a different class that uses `private sections = new Map<string, string[]>()` (`engine/ShaderBuilder.ts:30`) as a real plugin section registry exposed via `getSections`/`getAllSectionNames` (`engine/ShaderBuilder.ts:79-80`); it has no profiling block and is unaffected.

Recommended resolution: delete the block at `engine-gmt/engine/ShaderBuilder.ts:576-601`. The comment `// Section size profiling — log where the bytes are` (`engine-gmt/engine/ShaderBuilder.ts:576`) suggests it was a one-off diagnostic where the `console.log(profile)` line was removed but the setup was left behind. If profiling is wanted back, wire `profile` (and optionally `totalSize`) to a `console.debug` gated by a `DEBUG_SHADER_PROFILE` flag, or expose via a `lastBuildProfile` field on the builder so callers can inspect it after `buildFragment()`. Until then it costs ~25 lines and one O(n) pass per Main-variant build.

Confidence: high.
