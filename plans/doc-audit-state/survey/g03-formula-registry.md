## Public API surface

- `FractalRegistry` class ‚Äî instance exported as `registry` (`engine-gmt/engine/FractalRegistry.ts:34`). Methods: `register(def)`, `registerAlias(alias, targetId)`, `get(id)`, `getAll()` (dedupes aliases via `Set`), `getIds()`.
- `NodeRegistry` class ‚Äî instance exported as `nodeRegistry` (`engine-gmt/engine/NodeRegistry.ts:58`). Methods: `register(def)`, `get(id)`, `getAll()`, `getGrouped()` (returns `Record<category, string[]>`).
- `FractalDefinition` type at `engine-gmt/types/fractal.ts:67-118` ‚Äî public contract for every formula. Required: `id`, `name`, `shader.{function, loopBody}`, `parameters`, `defaultPreset`. Optional: `thumbnail`, `shortDescription`, `description`, `juliaType` ('julia'|'offset'|'none'), `tags`, `shader.{loopInit, getDist, preamble, preambleVars, usesSharedRotation, selfContainedSDE, supportsCuttingPlane}`, `flags.coordinateMode` ('Unified'|'DataAware'), `importSource` (Workshop re-edit metadata).
- `FractalParameter` type at `engine-gmt/types/fractal.ts:53-65` ‚Äî slot-keyed (paramA-F, vec2A-C, vec3A-C, vec4A-C) parameter spec with `type`, `min/max/step/default`, `scale` ('linear'|'log'|'pi'), `mode` ('rotation'|'direction'|'axes'|'toggle'|'mixed'), `options`, `linkable`. The `parameters` array can contain `null` to skip a slot or `[null, null, null, null]` (Modular placeholder).
- `NodeDefinition` type at `engine-gmt/engine/NodeRegistry.ts:23-31` ‚Äî used by the Modular graph builder. Has `id` (NodeType union), `label`, `category` (one of 7 fixed values: Fractals, Transforms, Folds, Primitives, Combiners (CSG), Utils, Distortion), `inputs: NodeInput[]`, `glsl: (ctx) => string`. `GLSLContext` provides `varName`, `in1`, `in2`, `getParam`, `indent`.
- `PipelineNode` / `GraphNode` / `FractalGraph` / `GraphEdge` types at `engine-gmt/types/graph.ts:10-43` ‚Äî Modular formula serialization shape.
- `Preset` type at `engine-gmt/types/fractal.ts:6-51` ‚Äî serialization shape; `features: Record<string, any>` holds all DDFS state.
- `FormulaType` union at `engine-gmt/types/common.ts:4` ‚Äî 43 string literals (42 formula IDs + Modular). Note: includes `'PseudoKleinian06'` which appears in unions but is an alias for the formula exported as `PseudoKleinian06` from `PseudoKleinianAdv.ts`.
- Formula registration: `engine-gmt/formulas/index.ts:50-104` builds `formulas` array in UI-display order, batch-calls `registry.register`, then registers 5 legacy aliases via `registry.registerAlias` (lines 107-111). Final export `Formulas = registry` (line 116) is re-exported as the public facade.
- Category data: `engine-gmt/formulas/categories.ts:7-99` ‚Äî `PREDEFINED_CATEGORIES` is a frozen `const` array of `{name, match: string[]}`. Uses string literal IDs to break import chain (commented at `formulas/index.ts:113-114`).

## Architecture

- **Two-registry model**: `FractalRegistry` holds whole-formula definitions (one per `.ts` in `formulas/`); `NodeRegistry` holds Modular graph node definitions (registered elsewhere ‚Äî likely under `engine-gmt/features/modular/` or graph compiler, owned by g09).
- **Singleton pattern**: Both registries are module-level singletons (`export const registry = new FractalRegistry()`). Registration happens at module import time via `formulas/index.ts` side-effects.
- **Alias support**: Aliases share storage with their target (`set(alias, def)` reuses the same `FractalDefinition` object). `getAll()` dedupes with `Set(values())`. Legacy aliases registered: `UberMenger‚ÜíMengerAdvanced`, `FoldingBrot‚ÜíBoxBulb`, `HyperTorus‚ÜíMandelorus`, `HyperbolicMandelbrot‚ÜíMandelBolic`, `RhombicIcosahedron‚ÜíCoxeter`.
- **Formula file shape (Tier B confirmed)**: Each formula is a single `.ts` that imports `FractalDefinition` from `'../types'` and exports a named const matching the `id`. The five mandatory parts:
  1. Metadata (`id`, `name`, `shortDescription`, `description`, `juliaType`, optional `tags`).
  2. `shader` block ‚Äî GLSL function string + `loopBody` invocation. Larger formulas add `loopInit`, `preamble`, `preambleVars`, `getDist`, and flag fields.
  3. `parameters: (FractalParameter | null)[]` ‚Äî slot-keyed, UI-driven.
  4. `defaultPreset` ‚Äî formula-scoped initial Zustand state (`features.coreMath`, `features.coloring`, lights, camera pose, etc.).
  5. Side-effect-free: registration happens only in `formulas/index.ts`, not in formula files themselves.
- **Shader composition contract**: Engine generates the iteration loop; formulas supply just the body. `function` is dropped at GLSL top scope; `loopBody` is spliced inside the iteration loop. `loopInit` runs before the loop. `preamble` declares globals/helpers above `function`. The Mandelbulb (`Mandelbulb.ts:12-67`) is the minimal shape: function + loopBody only.
- **Modular formula is a thin shell**: `Modular.ts:13-18` has empty shader strings + `getDist: ''` ‚Äî `ShaderFactory` intercepts the `'Modular'` ID and injects graph-compiled GLSL. `parameters: [null, null, null, null]` (line 21) is the placeholder marker ‚Äî parameters come from the graph nodes' `bindings` (covered by g09).
- **`juliaType` semantics**: `'julia'` = c is iteration constant (Mandelbulb classic), `'offset'` = c adds translation (AmazingBox), `'none'` = formula ignores c. Drives UI label only ‚Äî runtime behavior comes from the `coreMath.juliaMode` feature flag + shader code.
- **Shared rotation contract (`usesSharedRotation: true`)** at `engine-gmt/types/fractal.ts:87`: Formulas reading/writing `gmt_rotAxis/rotCos/rotSin` need the engine to swap state during interlace. 13 formulas opt in (Buffalo, Claude, Mandelbar3D, MakinBrot, MixPinski, Octahedron, Icosahedron, Cuboctahedron, Coxeter, Dodecahedron, GreatStellatedDodecahedron, RhombicDodecahedron, RhombicTriacontahedron, SierpinskiTetrahedron, TruncatedIcosahedron ‚Äî most "polyhedral IFS" formulas).
- **Cutting-plane estimator (`supportsCuttingPlane: true`)** at `engine-gmt/types/fractal.ts:91-94`: Engine auto-declares `cp_dmin/cp_scale/cp_trap` globals + initializes in loopInit; formula writes during iteration. When `estimator=5` (Cutting Plane), engine's getDist returns `vec2(abs(cp_dmin), cp_trap)`. 12 formulas opt in: SierpinskiTetrahedron, MengerSponge, MengerAdvanced, Octahedron, Icosahedron, Dodecahedron, Cuboctahedron, Coxeter, RhombicDodecahedron, RhombicTriacontahedron, GreatStellatedDodecahedron, TruncatedIcosahedron.
- **Self-contained SDE (`selfContainedSDE: true`)** at `engine-gmt/types/fractal.ts:88-90`: Formula owns its full SDE; loopBody calls function once then breaks. Engine guards: `SKIP_PRE_BAILOUT`, no hybrid fold injection, no interlacing. Only 2 formulas: `JuliaMorph` (line 155), `MandelTerrain` (line 272).
- **Custom `getDist`**: 10 formulas override the engine's default distance estimator with formula-specific code: AmazingSurface, Apollonian, JuliaMorph, KaliBox, KleinianJos, KleinianMobius, MandelTerrain, MarbleMarcher, MixPinski, PseudoKleinianAdv, PseudoKleinianMod4 (also Modular as empty string).
- **Preamble + preambleVars**: For mutable globals declared in `preamble` (e.g. Claude's `uCl_n4`, `claude_Phi`, `claude_goldenAxis` at `Claude.ts:110`), the formula must list them in `preambleVars: string[]` so the interlace rewriter knows to rename them when two formulas share GLSL. Only the Claude formula uses this currently (confirmed via Grep ‚Äî single hit). This was the contract introduced for the interlace system (per existing doc 24).
- **`tags` field**: 8 formulas have tags for filter/search: Apollonian (`sphere-packing, inversion, foam`), Cuboctahedron (`archimedean, ifs, knighty`), GreatStellatedDodecahedron, KleinianJos, KleinianMobius, PseudoKleinianAdv, PseudoKleinianMod4, TruncatedIcosahedron. Most older formulas have no tags.
- **`flags.coordinateMode`**: Declared in the type but **never used in any formula file** (grep across `engine-gmt/` returns only `types/fractal.ts`). Either reserved for future use or dead.
- **`importSource`**: Declared in the type, used only by `FormulaSelect.tsx` + `FormulaWorkshop.tsx` UI (not by any built-in formula file). Populated at runtime when a user imports a Fragmentarium-style `.frag` via the Workshop.
- **UI ordering**: `formulas/index.ts:50-101` ordering drives the loading screen (Featured / Others / Hybrids / System sections separated by comments). Categories from `categories.ts` are a separate filter UI layer that may overlap (formulas appear in multiple categories).
- **Bundle-splitting note**: `categories.ts` uses string literal IDs deliberately (`formulas/index.ts:113-114` comment) so the categories module doesn't drag every formula's GLSL into the entry chunk.
- **Outlier formulas (Tier C size scan)**:
  - **Largest** (>500 lines): `Mandelorus.ts` (593), `MandelTerrain.ts` (593), `MandelMap.ts` (588), `Borromean.ts` (552), `Appell.ts` (545). These are sprawling because of long GLSL helper functions, multi-mode projection logic (MandelMap), or terrain noise + multi-octave shading (MandelTerrain).
  - **Smallest** (<100 lines, "thin"): `Modular.ts` (35 ‚Äî placeholder for graph builder), `Kleinian.ts` (89), `MakinBrot.ts` (95), `AmazingSurf.ts` (97), `Mandelbar3D.ts` (97), `Tetrabrot.ts` (98).
  - **Structural outliers**: `Modular.ts` (empty shader strings, dynamic injection), `JuliaMorph.ts` + `MandelTerrain.ts` (selfContainedSDE), `Claude.ts` (only formula using `preambleVars` currently).

## Invariants and gotchas

- **Singleton registration is side-effectful** ‚Äî importing `engine-gmt/formulas/index.ts` causes 42 calls to `registry.register` + 5 alias calls. The `Formulas = registry` re-export is only valid after this import has executed; consumers must import from `formulas/index.ts`, not directly from `engine/FractalRegistry.ts`.
- **`getAll()` deduplicates aliases via `Set(values())`** ‚Äî aliases share the same `FractalDefinition` reference, so identity comparison works. Adding a new alias that points to a definition not yet registered logs a warning and silently fails (`FractalRegistry.ts:13-17`).
- **`FormulaType` union is hand-maintained** at `types/common.ts:4`. Adding a formula requires updating this union manually; the registry doesn't enforce membership. `getIds()` casts via `as FormulaType[]` (line 30) ‚Äî unsafe if a non-listed ID is registered.
- **`parameters` array slot order matters for UI**. `null` skips a slot. Engine maps parameters to uniform slots (paramA‚ÜíuParamA, vec2A‚ÜíuVec2A, etc.) by `id`, not position ‚Äî but UI iterates in array order.
- **`preambleVars` must list every mutable global** declared in `preamble`. Missing entries cause interlace rendering to share state across two formulas (silently corrupted output). The contract is documented in existing doc 24, but enforcement is only at runtime via the interlace test sweep.
- **`usesSharedRotation` is required for formulas calling `gmt_precalcRodrigues`** in their `loopInit`. Without it, interlace mode reads stale rotation state.
- **`selfContainedSDE` is mutually exclusive with hybrid-box and interlace** ‚Äî engine sets `SKIP_PRE_BAILOUT` and skips injection. JuliaMorph + MandelTerrain own their full SDE.
- **`Modular` has empty shader strings**. Any direct shader-string consumer must guard against this (the `id === 'Modular'` intercept lives in ShaderFactory, owned by g04/g09).
- **`FormulaType` includes `'PseudoKleinian06'`** but the file is `PseudoKleinianAdv.ts` exporting `PseudoKleinian06`. The filename does not match the type ID ‚Äî a confusing legacy. Import line at `formulas/index.ts:43` makes the mapping explicit.
- **`flags.coordinateMode` is dead** ‚Äî declared in type but never set in any formula file. Either remove or document the intended consumer.

## Open questions

- Orphan-sweep candidate: `engine-gmt/types/fractal.ts:99-101` `flags.coordinateMode` ‚Äî declared but no formula sets it and grep finds zero consumers outside the type file. Either remove or wire up.
- The `FormulaType` union at `types/common.ts:4` is hand-maintained alongside `formulas/index.ts` registrations ‚Äî single source-of-truth would prevent drift (e.g. via generated type from the formulas array).
- `NodeRegistry` is declared but the actual node registration site (where the graph nodes register themselves) lives outside this subsystem ‚Äî covered by g09 (Modular graph compiler).
