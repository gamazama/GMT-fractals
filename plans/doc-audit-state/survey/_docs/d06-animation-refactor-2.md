Targets shipping before `09_CANVAS_GRAPH_PROMPT.md` so canvas validation is unambiguous.
Key decisions: median preferred over mean (per-commit cost skewed by outliers); cap samples at 1000 for memory; diff script must handle absent `medianMs` for backward compat with `spike-pre.json` / `probe-pre.json`.
Preservable: rationale for why commit count alone misleads; the "metric trap" lesson reused throughout subsequent canvas work; pinned JSON shape extension contract.

## 11_CANVAS_GRAPH_REPORT.md

Shipped with a dramatic user-visible win at heavy seed (9000 keyframes / 6 tracks): `workerFps` 22→43 (+91%), `mainFps p50` 20→60 (vsync, +199%), longTasks 12→0, 1054 ms blocking → 0. The win is in worker/mainFps/longTasks instead.
Key decisions: `keyframesRef` (the `Keyframe[]` array reference) is the version token via clone-on-write writers in `sequenceSlice` — no counter needed; soft-selection mask uses xor-folded djb2 hash of sorted selected ids; `panX` baked into viewKey (rebuild-on-pan) instead of `ctx.translate` (the prompt's preferred approach) — the simpler tradeoff documented inline for future revisit; `drawGraph` calls drop to ZERO during graph-play after the overlay split (back useEffect doesn't fire); cache hit rate 100% on default seed.
Preservable: the metric-trap lesson (per-commit ms doesn't track user-felt smoothness; workerFps/longTasks do) is the load-bearing finding reused by every subsequent canvas work; cache-key contract; the 5 surprises (polyline wasn't the bottleneck — diamond drawing was; cache 100% hit ≠ bench moves; user-visible win didn't show in per-commit ms; panX-in-viewKey deviation rationale; soft-mask shipped without bench coverage); the follow-ons list (DPR, pan-translation cache, GraphSidebar LiveValueDisplay collapse, AnimationDocument deferred, soft-selection bench scenario).
MAY BE STALE: none — this is a foundational empirical finding; the follow-ons it lists are tracked / partly addressed by 16_, 17_, 20_ but the report itself remains the canonical record of what shipped.
