- **Proposed text:**
```
> Read every file in `<files>` top-to-bottom. To record current blob SHAs, run `node plans/doc-audit-state/scripts/blob-sha.mjs <path1> <path2> …` — the script accepts one or more paths and prints `<path><TAB><sha>` per line (single-arg form still works and prints the bare sha). Produce a survey doc at `plans/doc-audit-state/survey/<id>.md` with this exact shape:
```
- **Why:** addresses f-004. The orchestrator extends `blob-sha.mjs` so single-arg invocations remain unchanged but multi-arg invocations work and emit a path:sha map, removing the first-batch trap. The brief change makes the new usage discoverable.
- **Companion script change** — `plans/doc-audit-state/scripts/blob-sha.mjs` should be rewritten so that the arg-parse block (`const arg = process.argv[2]; if (!arg) { … }`) becomes a loop over `process.argv.slice(2)`. Single-arg behaviour: print bare sha (current). Multi-arg behaviour: print `<path><TAB><sha>` per line. Exit 1 if zero args supplied. Suggested implementation:
```js
const args = process.argv.slice(2);
if (args.length === 0) {
  process.stderr.write('usage: blob-sha.mjs <path> [<path> ...]\n');
  process.exit(1);
}
const single = args.length === 1;
for (const arg of args) {
  const abs = resolve(arg);
  if (!existsSync(abs)) {
    process.stderr.write(`blob-sha: not found: ${arg}\n`);
    process.exit(1);
  }
  try {
    const out = execFileSync('git', ['hash-object', '--', abs], { cwd: REPO_ROOT, encoding: 'utf8' }).trim();
    if (single) process.stdout.write(out + '\n');
    else process.stdout.write(`${arg}\t${out}\n`);
  } catch (e) {
    process.stderr.write(`blob-sha: ${e.message}\n`);
    process.exit(1);
  }
}
```
- **Risk:** existing surveys recorded SHAs correctly with single-arg calls, so backward compat is preserved. If an agent passes a glob (`**/*.ts`) expecting shell expansion, that will still fail on Windows PowerShell — but that failure mode pre-existed and is out of scope. Worst case: agents call the multi-arg form and get a slightly different output format than they expected; the format is self-describing (TAB-separated) and easy to parse.
