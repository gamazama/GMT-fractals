# Requirements: Vector Uniform Support (vec2/vec3) in GLSL Engine

**Status:** ✅ Implemented (v0.8.5)  
**Priority:** High  
**Impact:** Core Engine Architecture  
**Related:** Fragmentarium Importer, FeatureSystem, Animation Engine  
**Implementation Date:** 2026-02-27

---

## 1. Executive Summary

GMT now supports native vec2 and vec3 uniforms in addition to the existing 6 scalar parameters (`uParamA` through `uParamF`). This expands formula parameter capacity from 6 values to up to 18 values (6 scalars + 3 vec2 + 3 vec3).

### Available Uniforms

| Uniform | Type | Values |
|---------|------|--------|
| `uParamA` - `uParamF` | float | 6 scalar values |
| `uVec2A` - `uVec2C` | vec2 | 6 values (3 × vec2) |
| `uVec3A` - `uVec3C` | vec3 | 9 values (3 × vec3) |
| **Total Capacity** | | **21 values** |

---

## 2. Motivation

### 2.1 Current Limitation
```glsl
// Current approach - limited to 6 scalars
float Scale = uParamA;
float MinRad2 = uParamB;
float TransX = uParamC;  // vec3 split across 3 params
float TransY = uParamD;
float TransZ = uParamE;
float RotAngle = uParamF;
// No more params available!
```

### 2.2 Desired Capability
```glsl
// With vec3 support - 6 vec3s = 18 scalar values
float Scale = uParamA;
float MinRad2 = uParamB;
vec3 Trans = uVec3A;      // 3 values in 1 uniform
vec3 PreTranslation = uVec3B;  // 3 more values
vec2 FoldValues = uVec2A; // 2 values
float RotAngle = uParamC;
// Still have room for more!
```

### 2.3 Use Cases
- **Fragmentarium Import:** Direct mapping of vec3 uniforms (Trans, PreTranslation, JuliaValues, etc.)
- **Complex Formulas:** Formulas with multiple vector parameters
- **Color Parameters:** vec3 for RGB colors without consuming 3 scalar slots
- **Positional Offsets:** Multiple vec3 offsets for advanced folding

## 3. Requirements

### 3.1 Uniform Schema Extensions

**REQ-1:** Add new uniform types to `engine/UniformSchema.ts`:
```typescript
export enum Uniforms {
    // Existing scalar params
    ParamA = 'uParamA',
    ParamB = 'uParamB',
    ParamC = 'uParamC',
    ParamD = 'uParamD',
    ParamE = 'uParamE',
    ParamF = 'uParamF',
    
    // NEW: Vector uniforms
    Vec2A = 'uVec2A',
    Vec2B = 'uVec2B',
    Vec2C = 'uVec2C',
    Vec3A = 'uVec3A',
    Vec3B = 'uVec3B',
    Vec3C = 'uVec3C',
}
```

**REQ-2:** Define uniform metadata:
```typescript
export interface UniformDefinition {
    name: string;
    type: 'float' | 'int' | 'vec2' | 'vec3' | 'vec4' | 'mat3' | 'mat4';
    default: any;
    // ... existing fields
}
```

### 3.2 FeatureSystem Extensions

**REQ-3:** Extend `ParamConfig` to support vector uniforms:
```typescript
export interface ParamConfig {
    type: 'float' | 'int' | 'vec2' | 'vec3' | 'vec4' | ...;
    uniform?: string;  // Can now reference 'uVec3A', etc.
    default: number | number[] | THREE.Vector2 | THREE.Vector3;
    // For vec3: default is THREE.Vector3
    // For vec2: default is THREE.Vector2 or [x, y]
}
```

**REQ-4:** Update `ParamType` union type to include vector types:
```typescript
export type ParamType = 
    | 'float' 
    | 'int' 
    | 'vec2' 
    | 'vec3' 
    | 'vec4' 
    | 'color' 
    | 'boolean' 
    | ...;
```

### 3.3 ShaderBuilder Integration

**REQ-5:** Generate vec2/vec3 uniform declarations in shaders:
```glsl
// Generated by ShaderBuilder
uniform float uParamA;
uniform float uParamB;
uniform vec2 uVec2A;   // NEW
uniform vec3 uVec3A;   // NEW
uniform vec3 uVec3B;   // NEW
```

**REQ-6:** Handle vector uniform updates in render loop:
- THREE.js uniform updates for vec2/vec3 types
- Proper type conversion from store values

### 3.4 Animation Engine Support

**REQ-7:** Extend animation tracks to support vector interpolation:
```typescript
// Keyframe data for vec3
interface Vector3Keyframe {
    time: number;
    value: THREE.Vector3;  // or { x, y, z }

// Interpolation
function lerpVector3(a: THREE.Vector3, b: THREE.Vector3, t: number): THREE.Vector3
```

**REQ-8:** Update track system to handle vector types:
- Track type: 'vec2' | 'vec3'
- Vector interpolation in AnimationEngine
- Proper keyframe recording for vector params

### 3.5 UI Integration

**REQ-9:** AutoFeaturePanel already supports vec2/vec3 rendering (verified):
- `Vector2Pad` for vec2 types
- `Vector3Input` for vec3 types
- Just needs proper config from FeatureSystem

**REQ-10:** Parameter selector/mapping UI updates:
- Show vec2/vec3 options in mapping dropdown
- Visual indication of vector uniforms

### 3.6 Backward Compatibility

**REQ-11:** Existing formulas must continue to work:
- Current 6 scalar params remain unchanged
- Migration path for formulas wanting to use vectors
- Optional opt-in for vector uniforms

**REQ-12:** Store schema compatibility:
- Existing presets with scalar params still load

## 5. Technical Design

### 5.1 Uniform Slots Layout

**Proposed Layout:**
| Slot | Type | GLSL Name | Use Case |
|------|------|-----------|----------|
| 1 | float | uParamA | Primary scalar (Scale, Power) |
| 2 | float | uParamB | Secondary scalar (MinRad2) |
| 3 | vec2 | uVec2A | 2D values (FoldValues) |
| 4 | vec3 | uVec3A | Position/Translation |
| 5 | vec3 | uVec3B | Color/Offset |
| 6 | vec3 | uVec3C | Additional vector |
| 7 | float | uParamC | Remaining scalars |
| 8 | float | uParamD | Remaining scalars |
| 9 | float | uParamE | Remaining scalars |
| 10 | float | uParamF | Remaining scalars |

**Alternative (Mixed):**
| Slot | Type | GLSL Name |
|------|------|-----------|
| 1 | float | uParamA |
| 2 | float | uParamB |
| 3 | float | uParamC |
| 4 | vec2 | uVec2A |
| 5 | vec3 | uVec3A |
| 6 | float | uParamD |
| ... | ... | ... |

### 5.2 State Management

**Store Update:**
```typescript
interface CoreMathState {
    // Existing scalars
    paramA: number;
    paramB: number;
    // ...
    
    // NEW: Vector params
    vec2A: { x: number; y: number };
    vec3A: { x: number; y: number; z: number };
    vec3B: THREE.Vector3;  // Can use THREE types
}
```

### 5.3 GLSL Integration

**Formula Example:**
```glsl
void formula_MyFormula(inout vec4 z, inout float dr, inout float trap, vec4 c) {
    vec3 p = z.xyz;
    
    // Scalar params
    float scale = uParamA;
    float minRad2 = uParamB;
    
    // Vector params - clean and readable!
    vec3 translation = uVec3A;
    vec2 foldValues = uVec2A;
    vec3 colorOffset = uVec3B;
    
    // Formula logic...
    p += translation;
    p.xy = abs(p.xy + foldValues);
    
    z.xyz = p;
}
```

## 6. Risks & Mitigation

| Risk | Impact | Mitigation |
|------|--------|------------|
| Breaking existing formulas | High | Keep scalar params unchanged; vector is additive |
| Animation complexity | Medium | Reuse existing Vector3 interpolation from THREE.js |
| Shader compilation errors | Medium | Extensive testing with various GPU drivers |
| Performance degradation | Low | Vector uniforms are more efficient than multiple scalars |
| Store migration | Medium | Default values for new vector params; graceful fallback |
